# 可视化接口程序产品需求说明书

# 1. 引言

## 1.1 产品背景

在当前多项目、多数据源、多报文格式的开发场景中，接口开发与格式转换往往需要大量编码工作，开发效率低、维护成本高，且对开发人员的技术门槛要求较高。为解决上述问题，开发一款可视化接口程序，实现接口转换配置、快速可视化接口开发的无码/低码操作，适配大部分项目的开发需求，降低接口开发门槛，提升开发效率与可维护性。

## 1.2 产品目标

- 核心目标：提供可视化操作界面，实现接口转换配置与接口快速开发，无需大量编码，降低开发门槛，适配80%以上常规项目的接口开发需求。

- 效率目标：将常规接口开发周期缩短50%以上，接口转换配置耗时控制在10分钟内/单接口。

- 安全目标：保障接口操作的数据安全性，留存操作痕迹，防止误操作导致的数据丢失或泄露。

- 性能目标：支持高并发查询场景，常规查询响应时间≤300ms，增删改操作响应时间≤500ms，统计数据实时更新。

- 拓展目标：支持多种报文格式、多种数据库适配，可灵活拓展新的报文类型与数据库类型。

## 适用范围

本产品适用于各类需要进行接口开发、接口格式转换的项目，涵盖后端接口开发、第三方接口适配、数据交互接口搭建等场景，适配中小规模项目及大型项目的常规接口开发需求，面向开发人员、测试人员、运维人员使用。

## 术语定义

- 报文格式转换：将一种格式的请求/响应报文（如JSON、XML）转换为另一种目标格式，同时完成字段的映射与加工。

- 可视化接口开发：通过拖拽、选择等可视化操作，无需编写SQL语句与接口代码，快速完成数据库增删改查接口的开发。

- 缓存数据库：用于存储表结构、常用查询结果等数据，提升查询效率，本文特指本地缓存与分布式缓存结合的缓存体系。

- 字段加工：对接口字段或数据库字段进行截位、补齐、格式转换、运算等处理，满足业务需求。

- 多表查询可视化：通过可视化选择表、主键关联关系、返回字段，自动生成多表关联查询逻辑。

# 2. 产品概述

## 2.1 产品定位

一款无码/低码的可视化接口工具，聚焦接口转换配置与快速接口开发两大核心场景，提供简洁易用的可视化操作界面，适配大部分项目的技术栈，降低接口开发与维护成本，提升开发效率，同时保障数据安全与系统性能。

## 核心功能模块

本产品分为两大核心模块，各模块功能相互独立且可联动，具体如下：

- 模块一：接口转换配置——实现报文格式转换、字段映射配置、字段加工等功能，支持多种报文格式，配置化操作，无需编码。

- 模块二：快速可视化接口开发——支持多数据库连接、表结构可视化、数据库增删改查接口可视化开发、缓存优化、数据安全与性能统计等功能。

# 3. 核心功能需求（详细）

## 模块一：接口转换配置

本模块核心目标是实现接口报文的格式转换、字段映射与字段加工，配置化操作，无需编写转换代码，支持多种报文格式，适配不同接口对接场景。同时补充实现逻辑，明确配置存储形式与程序调用机制，确保转换流程可追溯、可复用。

### 3.1.1 报文格式转换（手工选择）

- 支持的报文格式：JSON、XML、CSV、FormData（四大主流格式，可拓展其他格式）。

- 操作方式：可视化界面手工选择“源报文格式”和“目标报文格式”，支持双向转换（如JSON→XML、XML→JSON等）。

- 转换规则：
        

    - 自动匹配同源字段：若源报文与目标报文的字段名、层级一致，自动完成字段映射，无需手动配置。

    - 格式校验：转换后自动校验目标报文格式的合法性（如JSON语法校验、XML标签闭合校验），校验失败提示具体错误位置。

    - 转换预览：配置完成后，可输入测试源报文，实时预览转换后的目标报文，支持修改测试报文重新预览。

- 配置保存：支持将转换配置（源格式、目标格式、字段映射规则、加工规则）保存为模板，后续可直接复用，支持模板的查询、编辑、删除。

### 3.1.2 字段转换（配置化，支持固定值填入）

- 字段映射配置：可视化界面展示源报文字段（树形结构）与目标报文字段（树形结构），支持拖拽或下拉选择完成字段映射。

- 固定值配置：支持为目标报文的字段填入固定值（支持字符串、数字、日期、布尔值等类型），固定值优先级高于源字段映射（若配置固定值，忽略源字段的值）。

- 字段新增：支持在目标报文中新增字段，新增字段可选择“映射源字段”“填入固定值”“字段加工生成”三种方式。

- 字段删除/编辑：支持删除目标报文中无需的字段，编辑字段的名称、类型、映射规则或固定值。

- 批量操作：支持批量选择源字段，批量映射到目标字段（字段名一致时），支持批量清除映射规则。

### 3.1.3 字段加工（简单加工）

在字段转换过程中，支持对源字段或目标字段进行简单加工，加工规则可配置，支持单个字段加工或批量字段加工，加工后的值作为目标字段的最终值。

- 支持的加工类型：
        

    - 截位：左截、右截、中间截，可配置截位长度（如截取字符串前10位、后5位）。

    - 补齐：左补齐、右补齐，可配置补齐字符（如0、空格）和补齐后总长度（如将数字补齐为8位，不足补0）。

    - 格式转换：字符串→数字、数字→字符串、日期格式转换（如yyyy-MM-dd→MM/dd/yyyy）、布尔值转换（如true→1、false→0）。

    - 大小写转换：全部大写、全部小写、首字母大写。

    - 去空格：去除字符串前后空格、去除所有空格。

- 加工配置：选择需要加工的字段，选择加工类型，配置加工参数（如截位长度、补齐字符），支持预览加工后的结果。

- 多加工规则叠加：支持为单个字段配置多个加工规则（如先去空格，再截位，最后转换为大写），按配置顺序执行加工。

### 3.1.4 报文特定字段识别与渠道模板配置

支持识别报文中的特定字段（如渠道字段），基于该字段适配不同渠道的处理要求，实现不同渠道对应不同报文模板的选择与配置，提升接口适配的灵活性。

- 特定字段识别：可可视化配置需要识别的特定字段（如“channel”“渠道编码”等），支持自定义字段名称，系统自动解析报文（所有支持的报文格式），提取该特定字段的值，识别准确率≥99%。

- 渠道适配：预设常见渠道类型（可手动新增、编辑、删除渠道），支持将识别到的特定字段值与渠道进行关联（如字段值“APP”关联“移动端APP渠道”，字段值“WEB”关联“网页端渠道”）。

- 模板配置与选择：为不同渠道单独配置对应的报文转换模板（复用3.1.1-3.1.3的格式转换、字段映射、字段加工规则），支持两种模板使用方式：自动匹配：系统识别到特定字段后，自动匹配该字段对应的渠道模板，无需手动选择，直接执行对应的报文转换逻辑。

- 手动选择：支持手动选择渠道，进而选择该渠道对应的报文模板，适配特殊场景下的灵活调整。

- 模板管理：支持按渠道分类管理报文模板，可查看各渠道关联的模板、编辑模板规则、替换渠道对应的模板，支持模板复制（快速创建同类渠道模板）。

- 去渠道适配：基于识别到的渠道字段，可配置该渠道专属的去渠道处理规则（如字段过滤、格式调整），确保不同渠道的报文输出符合对应业务要求。

- 端口绑定：每个渠道除关联转换模板外，还可绑定一个目标端口地址，用于将转换后的报文转发到对应系统。端口绑定配置详见3.1.5。

### 3.1.5 端口分发路由配置

支持将转换后的目标报文按渠道路由到对应系统的指定端口，完成”报文转换 → 端口转发 → 响应返回”全流程，无需额外编码开发。端口分发与报文转换共用同一套渠道识别逻辑，转接口与接口转换的处理流程一致。

**核心设计：两字段路由表**

每条端口路由记录包含两个关键字段：

- `channel_code`（渠道标识）：对应渠道配置中的渠道编码，系统通过识别报文中的特定字段值匹配到该渠道后，自动查找绑定的端口地址。
- `port_address`（端口地址）：目标系统的完整访问地址（含协议、IP/域名、端口号及路径，如 `http://10.0.0.1:8081/api/receive`），转换后的报文将发送至此地址。

**端口路由配置项：**

- 系统名称：目标系统的描述名称，便于识别管理。
- 端口地址（`port_address`）：目标端口完整URL，支持HTTP/HTTPS。
- 请求方式（`port_method`）：GET / POST / PUT / DELETE，默认POST。
- 连接超时（`timeout`）：默认3000ms，可配置。
- 重试次数（`retry_count`）：默认3次，失败自动重试，重试耗尽后记录日志并返回错误信息。
- 连通性测试：配置保存前可测试端口是否可达。
- 请求转换模板（`request_template_id`）：绑定一个转换模板，用于将 A 系统报文转换为 B 系统所需格式，复用3.1.1-3.1.3的格式转换、字段映射、字段加工能力。
- 应答转换模板（`response_template_id`）：绑定一个转换模板，用于将 B 系统的应答报文转换为 A 系统所需格式；若不配置则直接透传 B 的原始应答。

**完整双向调用流程：**

```
A 系统发来报文
  ↓ 识别渠道字段，匹配 channel_code
  ↓ [请求转换模板] 格式转换 → 字段映射 → 字段加工
转发给 B 系统（port_address）
  ↓ B 系统返回应答报文
  ↓ [应答转换模板] 格式转换 → 字段映射 → 字段加工（按 A 系统格式要求）
返回给 A 系统
```

具体步骤：
1. 接收 A 系统源报文，解析识别字段，匹配渠道（`channel_code`）；
2. 按渠道找到绑定的请求转换模板（`request_template_id`），执行格式转换 → 字段映射 → 字段加工；
3. 按渠道找到 `port_address`，通过 HTTP 将转换后的报文发送至 B 系统；
4. 接收 B 系统应答报文，校验格式合法性；校验失败触发重试；
5. 若配置了应答转换模板（`response_template_id`），对 B 系统应答执行格式转换 → 字段映射 → 字段加工，转换为 A 系统所需格式；
6. 将最终应答返回给 A 系统，同步记录完整调用日志（含请求报文、B系统应答原文、最终返回报文、耗时）。

**配置保存与复用：** 端口路由配置以渠道为维度管理，支持新增、编辑、删除、连通性测试；请求模板与应答模板均复用现有转换模板管理（3.1.5），一条路由记录即可描述完整的双向转换链路，后续通过 `POST /api/dispatch` 一键触发全流程。

### 3.1.6 接口转换实现逻辑（配置存储+程序调用）

- 配置存储形式：所有接口转换相关配置（报文格式、字段映射规则、字段加工规则、渠道模板、转换模板、端口路由等），均以JSON格式存储在单独的MySQL配置数据库中，每条配置关联唯一ID、创建人、创建时间、修改时间；其中，常用的转换模板和渠道路由配置可同步缓存至Redis，提升调用效率；配置数据支持版本管理，修改后留存历史版本，可回溯恢复。

- 程序调用机制（两种调用模式）：
  - **仅转换模式**（`POST /api/convert`）：接收源报文和模板ID，执行”格式解析 → 字段映射 → 字段加工”，返回转换后的目标报文，不做端口转发。
  - **转换分发模式**（`POST /api/dispatch`）：接收源报文，自动识别渠道字段，执行完整链路”渠道识别 → 格式转换 → 字段映射 → 字段加工 → 端口转发（按 `channel_code` 查 `port_address`）→ 接收响应 → 返回”；两种模式共用同一套转换逻辑，转接口与接口转换处理一致。
  - 若配置了日志开关，同步记录转换日志（含配置ID、源报文、目标报文、操作人、耗时），纳入日志管理范围。

## 模块二：快速可视化接口开发

本模块核心目标是通过可视化操作，快速完成数据库接口的开发，支持多数据库连接、表结构可视化、增删改查操作配置，同时保障数据安全与系统性能，支持高并发场景适配。补充实现逻辑，明确接口配置的存储形式与程序调用机制，确保接口开发后可正常调用、灵活维护。

### 3.2.1 多数据库连接支持

- 支持的数据库类型：MySQL 8.0+、Oracle 11g/12c、PostgreSQL 12+（可通过插件拓展其他关系型数据库）。

- 连接配置：可视化界面配置数据库连接信息（驱动类、URL、用户名、密码、连接池大小、超时时间），支持测试连接（验证连接是否成功）。

- 连接管理：支持保存多个数据库连接配置（区分环境：开发、测试、生产），支持连接的编辑、删除、切换，默认保存最近使用的3个连接。

- 连接权限：不同用户可配置不同的数据库连接权限（如部分用户仅可查看，不可操作），保障数据库安全。

### 3.2.2 表结构查询与多级菜单展示（缓存存储）

- 表结构查询：连接数据库后，自动查询该数据库下所有表的信息，包括表名、表注释、表字段（字段名、字段类型、字段注释、是否主键、是否非空、默认值）。

- 多级菜单展示：以树形多级菜单形式展示表结构，一级菜单为数据库名称，二级菜单为表名（括号显示表注释），三级菜单为表字段（括号显示字段注释、字段类型），支持展开/折叠。

- 缓存存储：查询到的表结构信息（表名、表注释、字段信息等）自动存储到Redis缓存数据库中，缓存有效期可配置（默认24小时），避免重复查询数据库，提升界面响应速度。

- 缓存更新：支持手动刷新缓存（当数据库表结构发生变化时，手动触发刷新，更新缓存中的表结构信息），支持设置缓存过期时间；支持表结构信息导出（Excel格式），便于查看与备份；暂不支持表结构信息导入（避免与数据库实际表结构冲突）。

### 3.2.3 缓存数据查询限制

- 缓存数据支持查询：可通过表名、字段名模糊查询缓存中的表结构信息，快速定位目标表和字段。

- 禁止自动可视化展示：缓存中的表名称、表字段不会自动在可视化开发界面中展示，需用户手动选择数据库连接，触发表结构查询后，才会展示对应的表结构（避免缓存数据与实际数据库表结构不一致）。

### 3.2.4 普通数据库增删改查支持

普通数据库（非缓存数据库）支持完整的增删改查操作，所有操作通过可视化配置完成，自动生成对应的SQL语句，无需手动编写。

#### 3.2.4.1 查询操作（支持多表查询）

- 单表查询：可视化选择目标数据库、目标表，勾选需要返回的字段，配置查询条件（等于、不等于、大于、小于、包含、不包含等），支持多条件组合（且/或）。

- 多表查询：
        

    - 可视化选择多个需要关联的表（支持2-5表关联），通过下拉选择主键关联关系（如A表主键id关联B表foreign_id），自动识别关联字段（主键与外键），支持手动调整关联关系。

    - 返回字段选择：可分别选择每个表中需要返回的字段，支持给字段重命名（避免字段名重复）。

- 字段加工：支持对返回的字段进行加工（同接口转换的字段加工类型：截位、补齐、格式转换等），加工规则配置后，查询结果自动展示加工后的值。

- 数据加工：支持对查询到的数据进行简单运算（加减乘除），可选择两个字段进行运算（如字段A+字段B、字段C-字段D），运算结果作为新的返回字段。

- 查询条件规范化：请求中的字段值可通过可视化选择，填入对应的查询条件中（如将请求报文中的“userId”字段值，填入查询条件“id等于”中），支持动态赋值，适配不同请求场景。

- 查询预览：配置完成后，可输入测试参数，预览查询结果，支持调整查询条件重新预览。

#### 3.2.4.2 插入数据操作

- 多表插入支持：支持同时插入多张表的数据（最多支持3张表），每张表插入1条数据，需分别配置每张表的插入字段与数据来源。

- 插入字段配置：可视化选择目标表，勾选需要插入的字段（支持全表字段勾选，也可选择部分字段），必须配置所有非空字段（若字段有默认值，可选择使用默认值）。

- 数据来源（3种可选，可混合使用）：
        

    - 请求报文：选择请求报文中的对应字段，将请求字段值作为插入值。

    - 自定义字段：配置固定的自定义值（如常量、日期、随机数），作为插入值。

    - 字段运算：支持表数据之间、请求字段与表数据之间的运算（如A表字段+请求字段、B表字段×固定值），运算结果作为插入值。

- 数据校验：插入前自动校验数据合法性（如字段类型匹配、非空字段是否有值、字段长度是否符合数据库要求），校验失败提示具体错误信息，不执行插入操作。

- 事务支持：多表插入时，开启事务，若其中一张表插入失败，所有表的插入操作回滚，保障数据完整性。

#### 3.2.4.3 删除数据操作

- 条件配置：可视化配置删除条件，支持多条件组合（且/或），条件必须唯一（如通过主键删除、唯一索引删除），避免批量删除误操作。

- 条件校验：删除前自动校验条件的唯一性，若条件不唯一（可能导致批量删除），提示用户确认，确认后才可执行删除操作。

- 删除预览：配置删除条件后，可预览将被删除的数据（仅展示前10条），确认无误后执行删除操作。

- 禁止批量删除：默认禁止无明确唯一条件的批量删除，若需批量删除，需手动勾选“允许批量删除”，并输入操作密码（二次验证）。

#### 3.2.4.4 修改数据操作

- 修改字段配置：可视化选择目标表、需要修改的字段，配置修改后的值（数据来源同插入操作：请求报文、自定义字段、字段运算）。

- 修改条件：必须配置唯一的修改条件（如主键、唯一索引），确保每次修改仅针对一条数据，避免批量修改误操作。

- 数据校验：修改前自动校验修改后的数据合法性（字段类型、长度、非空约束等），校验失败不执行修改操作。

- 修改预览：配置完成后，可预览修改前、修改后的数据对比，确认无误后执行修改操作。

### 3.2.5 数据安全（SQL留存）

- SQL留存范围：所有增删改操作生成的SQL语句，均需保存到单独的MySQL数据库中（与业务数据库分离），留存不可修改。

- 留存字段：SQL语句、操作类型（新增/删除/修改）、操作人、操作时间、操作IP、目标数据库、目标表、操作结果（成功/失败）、错误信息（若失败）。

- SQL查询：支持按操作人、操作时间、操作类型、目标表等条件查询留存的SQL语句，支持导出SQL记录（Excel格式），用于审计与问题排查；同时支持通过Excel导入SQL留存记录（仅支持管理员操作，用于数据迁移、备份恢复），导入时自动校验数据格式合法性。

- 留存期限：SQL记录默认留存1年，可配置留存期限（最长3年），过期自动清理（可配置是否备份后清理）。

### 3.2.11 日志管理

实现交易操作日志的打印、展示与开关控制，确保交易SQL、报错信息可追溯，同时支持通过菜单开关控制日志查询，降低不必要的性能消耗，兼顾可追溯性与系统性能。

- 日志打印要求：交易SQL日志：所有交易相关的数据库操作（增删改查）生成的SQL语句，均需打印到系统日志中，日志需包含SQL语句、操作人、操作时间、操作IP、目标库表等关键信息，与SQL留存数据保持一致。

- 报错信息日志：系统运行过程中出现的所有错误（如SQL执行失败、接口调用异常、配置错误等），需完整打印报错信息，包括错误代码、错误描述、出错位置、操作场景、操作人及时间，便于问题排查。

- 日志格式：日志格式统一规范，包含时间戳、日志级别（INFO/ERROR）、操作模块、关键信息，确保可读性，支持日志导出（TXT/Excel格式）。

- 日志管理菜单：管理平台新增“日志管理”专属菜单，与其他管理菜单并列，菜单支持手动开关（开启/关闭）。开启状态：菜单正常展示，支持按操作人、操作时间、日志级别、操作模块等条件查询日志信息，支持日志筛选、排序、导出（Excel/TXT格式），实时展示最新日志；同时支持通过Excel导入日志记录（仅管理员操作，用于日志备份恢复），导入时校验日志格式与字段完整性。

- 关闭状态：菜单隐藏，系统不再执行日志查询相关操作（包括后台查询、页面渲染），减少数据库查询和页面渲染压力，实现性能解耦。

- 性能优化：日志查询操作与核心业务操作分离，关闭日志菜单后，仅停止日志查询功能，不影响日志打印和存储；日志存储采用分级存储策略，避免日志量过大占用过多资源，日志默认留存30天，可配置留存期限。

### 3.2.6 性能统计（增删改查数据统计）

- 统计维度：按时间维度（小时、天、周、月）统计增删改查各操作的执行次数、成功次数、失败次数、平均响应时间、峰值响应时间。

- 统计展示：通过ECharts图表可视化展示统计数据（折线图、柱状图），支持切换时间维度、操作类型，支持查看具体时间段的统计详情。

- 异常告警：当某类操作的失败率超过5%、平均响应时间超过1000ms时，自动触发告警（页面提示、消息通知），支持配置告警阈值。

- 统计数据存储：统计数据默认存储到MySQL数据库，若数据量较大，可切换到Elasticsearch存储，支持数据定期清理（默认留存6个月）；支持统计数据导出（Excel格式），可按时间维度、操作类型导出对应统计结果，便于数据分析；暂不支持统计数据导入（避免手动修改统计结果，确保数据真实性）。

### 3.2.7 创建缓存查询（高并发适配）

- 缓存查询创建：对于常用的查询操作（如高频查询、查询条件固定的查询），支持创建本地缓存（Caffeine）+ 分布式缓存（Redis）双重缓存，提升查询效率。

- 缓存配置：可视化配置缓存key、缓存有效期（默认10分钟，可配置）、缓存更新策略（手动更新、定时更新、触发更新）。

- 缓存管理：支持查看所有缓存查询的列表（缓存key、缓存有效期、创建时间、命中次数），支持手动清除缓存、更新缓存，支持查看缓存命中情况统计；支持缓存查询配置导出（Excel格式），便于批量备份；支持通过Excel导入缓存查询配置（仅管理员操作），导入时自动校验配置合法性，避免无效配置。

- 高并发适配：缓存查询支持高并发访问，避免缓存穿透、缓存击穿（通过缓存预热、互斥锁、布隆过滤器实现，默认开启基础防护）。

### 3.2.8 常用字段公式（自定义复杂查询条件）

支持用户自定义复杂的查询条件公式，新增“常用字段公式管理”专属菜单，实现公式的集中管理，包含数据库、条件、接口字段关联等完整配置，支持复制新增功能，确保公式配置规范、复用便捷。

- 单独菜单管理：系统新增“常用字段公式管理”专属菜单，与分库分表配置、日志管理菜单并列，菜单内实现公式的新增、编辑、删除、查询、复制等全操作，界面布局与其他配置菜单保持一致，提升操作连贯性。

- 公式配置详情（含数据库、条件、接口字段关联）：
        

    - 基础信息配置：需填写公式名称（唯一）、所属业务场景（可分类）、备注（可选），便于后续查询和管理。

    - 数据库关联：可视化选择公式所关联的数据库（支持已配置的所有数据库类型），明确公式适用的数据源，避免跨数据库字段调用异常。

    - 条件配置：支持配置复杂查询条件，包括字段之间的加减乘除运算、逻辑运算（与/或/非）、比较运算（等于、大于、小于等），可选择数据库表字段、自定义常量作为条件参数，配置过程可视化，支持条件预览。

    - 接口字段关联：若公式涉及接口字段（如请求报文中的字段），需手动选择对应的接口（支持已创建的所有接口），关联接口后，可选择该接口下的具体字段作为公式的参数，确保接口字段调用的准确性，避免字段不匹配导致的公式失效。

- 复制新增功能：支持对已配置的公式进行复制操作，复制后生成一条与原公式完全一致的新公式，新公式需修改关键信息（至少修改公式名称、条件参数或关联字段中的一项）后，方可保存，避免重复配置且保障公式唯一性。

- 公式调用：在查询条件配置、增删改操作的数据来源配置中，可直接下拉选择已保存的常用公式，无需重复配置复杂条件，调用后可查看公式详情，支持根据当前场景微调公式参数（微调后不改变原公式）。

- 公式校验：公式配置（含复制修改后）完成后，系统自动校验公式逻辑合法性、数据库关联正确性、接口字段匹配性，校验失败提示具体错误信息（如“关联接口不存在”“字段类型不匹配”），校验通过后方可保存；支持常用字段公式导出（Excel格式），便于批量备份与分享；支持通过Excel导入常用字段公式（仅管理员操作），导入时自动校验公式逻辑与格式，导入成功后可直接复用。

### 3.2.9 拓展性（多报文格式处理）

- 报文格式支持：除基础的JSON、XML、CSV、FormData格式外，支持通过插件拓展其他报文格式（如SOAP、Protobuf），适配不同第三方接口对接需求。

- 可视化选择：所有支持的报文格式，均在可视化界面中可手工选择（下拉菜单），无需修改代码，拓展后自动同步到界面。

- 格式解析：拓展的报文格式需支持自动解析，解析后展示字段树形结构，支持字段映射与加工，与基础格式的操作逻辑一致。

### 3.2.10 分库分表操作支持

支持数据库分库分表操作，提供单独的菜单进行分库分表逻辑配置，接口开发时可直接选择已配置的逻辑，适配分库分表场景下的接口开发需求，核心目标是在创建SQL后准确定位到目标表，适配不同业务模块的差异化库表架构（如客户信息单库、交易信息多库多表等场景）。

- 单独菜单配置：系统新增“分库分表配置”专属菜单，与其他数据库配置菜单区分，支持分库分表逻辑的新增、编辑、删除、查询，界面操作与现有配置风格保持一致。

- 

- 分库分表逻辑配置（适配多库多表差异化场景）：
        

- 基础规则配置：核心逻辑为「业务模块区分+请求字段关联+库表映射」，先按业务模块（如客户信息、交易信息）划分库表组，每个模块可配置独立的库表架构；通过请求中的指定字段（如客户ID、交易ID），结合配置的规则，查询目标数据库表，准确定位数据归属的库和表。

- 库表映射配置：支持差异化架构配置，无需固定为“两个库16张表”，可按需配置各业务模块的库表数量，例如：客户信息模块配置1个客户库、若干张客户相关表；交易信息模块配置2个交易库，每个交易库下配置16张交易相关表；可分别配置每个库的名称、连接信息，每张表的表名、表注释、编号范围或关联规则，明确数据归属逻辑（如交易ID前缀为1-8对应交易库1，前缀为9-16对应交易库2）。

- 请求字段关联：可视化选择请求报文中的关联字段（如“userId”对应客户模块、“tradeId”对应交易模块），配置该字段与业务模块、库表的关联关系，确保系统能通过该字段，先匹配对应业务模块，再定位到具体的库和表，避免跨模块定位错误。

- 逻辑校验：配置完成后，可选择业务模块、输入测试请求字段值，预览分库分表判断结果（即对应归属的业务模块、库和表），校验逻辑准确性，避免配置错误导致SQL定位失败；支持批量测试多个请求字段值，快速验证规则适配性。

- 接口配置关联：在进行数据库增删改查接口配置时（尤其是多表操作、高频操作），可直接下拉选择已配置好的分库分表逻辑，无需重复配置库表判断规则，系统将自动按照配置的逻辑，定位目标库和表并执行对应操作。

- 逻辑管理：支持按业务模块对分库分表逻辑进行分类命名（如“客户数据分库分表”“交易数据分库分表”），便于快速查找和选择；支持复制逻辑配置，快速适配同类分库分表场景（可灵活调整库表数量，无需遵循固定架构）。

- 兼容性：分库分表操作与现有增删改查、缓存查询等功能兼容，执行操作时，系统自动适配分库分表逻辑，同时留存对应的SQL操作记录（区分库表），纳入数据安全统计范围。

- 3.2.13 可视化接口开发实现逻辑（配置存储+程序调用）
        

- 配置存储形式：所有可视化接口开发相关配置（数据库连接信息、表结构缓存、接口配置、分库分表逻辑、常用字段公式、缓存查询配置等），分类存储在配置数据库中：数据库连接信息加密存储（密码脱敏）；表结构信息缓存至Redis，同步备份至配置库；接口配置（含SQL模板、字段加工规则、条件配置等）以JSON格式存储，关联接口ID、所属业务模块、创建人；常用字段公式、分库分表逻辑单独存储，关联对应配置ID，支持复用。

- 程序调用机制：1. 接口配置完成并发布后，系统自动生成唯一接口访问路径，同步至Swagger接口文档；2. 外部系统/用户通过访问路径调用接口，传入请求参数（如查询条件、插入数据）；3. 程序接收请求后，根据接口ID读取对应的接口配置，解析配置中的数据库连接、表结构、SQL模板、分库分表逻辑（若有）；4. 程序根据配置自动生成SQL语句，若涉及分库分表，先通过请求字段定位目标库表，再执行SQL操作；5. 执行完成后，对返回数据进行字段加工（若有），校验数据合法性，返回接口响应结果（数据、执行状态、耗时）；6. 同步执行日志打印、SQL留存、性能统计操作，确保操作可追溯、可监控。
