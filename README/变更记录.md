### CHG-004 字段映射 srcField 支持点号路径访问 XML 嵌套字段

- **日期**：2026-03-27
- **影响单元**：M1-2（字段映射配置 — `srcField` 格式约定）、M1-6（报文转换调用接口 — `applyMapping` 实现）
- **变更类型**：范围新增（新增功能点）
- **变更前**：
  - `applyMapping()` 对 `srcField` 只做顶层平铺查找（`srcMap.containsKey(srcField)`）
  - XML 报文被解析为嵌套 Map（如 `{head: {FunctionId: "170350"}, body: {...}}`），`srcField: "FunctionId"` 无法命中 `head` 下的字段，仅能映射顶层 key（`head`、`body`）
- **变更后**：
  - `applyMapping()` 新增 `getNestedValue(map, path)` 方法，支持点号路径递归取值
  - `srcField` 可配置为多级路径，如 `head.FunctionId`、`body.OffSet`，无点号时退化为原有平铺查找（向后兼容）
  - `mapping_rule` 示例：`[{"srcField": "head.FunctionId", "targetField": "functionId"}]`
- **影响文件**：
  - `backend/src/main/java/com/powergateway/service/ConvertService.java`（`applyMapping` + 新增 `getNestedValue`）
  - `backend/src/test/java/com/powergateway/M16ConvertApiTest.java`（新增验收测试 `convert_xmlNestedPath_dotNotationMapping`）
- **需求文档更新**：
  - `需求拆分与最小实现方案.md` → M1-2 实现方案（`srcField` 格式说明）、M1-6 实现方案（`applyMapping` 逻辑）
  - `开发计划.md` → M1-6 行描述
- **原因**：XML 报文存在 `<root><head>...</head><body>...</body></root>` 层级结构，平铺查找无法访问嵌套字段，需支持路径配置

### CHG-003 修复字段映射/加工页面源字段不显示问题

- **日期**：2026-03-27
- **影响单元**：M1-2（字段映射配置）、M1-3（字段加工引擎前端）
- **变更类型**：Bug 修复（不改变对外行为）
- **根因**：`vue-draggable-next` v2.x 的 `render()` 函数仅读取 `$slots.default`，从不读取 `$slots.item`。代码使用的 `<template #item="{ element }">` 是 `vuedraggable` v4.x（官方包）的 API，在 `vue-draggable-next` 中该 slot 被完全忽略，组件渲染空 div，导致所有手动添加或解析报文得到的源字段均不显示。
- **变更前**：`FieldMapping.vue` 和 `FieldProcess.vue` 中 `<draggable>` 内使用 `<template #item="{ element }">` 渲染列表项
- **变更后**：改为 `vue-draggable-next` 兼容的 default slot + `v-for` 写法，每个列表项通过 `v-for="element in list"` 直接渲染
- **影响文件**：
  - `frontend/src/views/convert/FieldMapping.vue`（srcFields 和 mappingRules 两处 draggable）
  - `frontend/src/views/convert/FieldProcess.vue`（rules draggable）
