# PowerGateway 项目技术栈总结

> PowerGateway 是一个低代码/零代码可视化接口开发平台，核心价值：减少接口开发工时 50% 以上，单个接口转换配置 ~10 分钟内完成。

---

## 二、核心架构设计

### 2.1 三库分离

```
配置库（MySQL）   ← 存储模板、接口配置、用户信息（默认数据源）
业务库（动态）    ← 按 dbId 动态切换，支持 MySQL / Oracle / PostgreSQL
审计库（MySQL）   ← 独立 MySQL，@DS("audit") 注解，保留 1 年
```

业务库通过 `DynamicDataSourceContextHolder.push(dbId)` 在 Service 层动态切换，执行结束后 `pop`，与配置库完全隔离。

### 2.2 双层缓存 + 防击穿

```
请求 → Caffeine（本地，毫秒级）→ Redis（分布式）→ DB
                                      ↑
                          SET NX PX 3000 分布式锁（防缓存击穿）
```

- 查询目标响应时间 ≤ 300ms，增删改 ≤ 500ms
- 缓存 TTL 通过 `sys_config` 全局配置，支持热更新

### 2.3 配置驱动 SQL 生成

```
可视化配置 → config_json（持久化）→ QueryBuilder / InsertBuilder
                                    / UpdateBuilder / DeleteBuilder
                                          ↓
                                    动态 SQL → 目标数据库
```

所有接口 SQL 均在运行时由 Builder 动态构造，不存硬编码 SQL，配置即代码。

### 2.4 AOP 异步审计

```
主链路：Controller → Service → Executor → 响应
                                  ↓（AOP @Around 拦截）
                           LinkedBlockingQueue
                                  ↓（@Async 异步）
                            审计库 sql_audit_log
```

三套 AOP 切面（`@AuditLog` / `@SysLogRecord` / `@PerfStat`）均异步写入，对主链路零影响。

### 2.5 多格式报文互转

```
请求报文 → 格式解析 → 统一 Map<String, Object>
                              ↓
                       字段映射（拖拽配置）
                              ↓
                       字段加工（策略模式）
                              ↓
                       序列化为目标格式 → 输出报文
```

支持 JSON ↔ XML ↔ CSV ↔ FormData 任意互转，可扩展 SOAP、Protobuf。

### 2.6 接口状态机

```
draft（草稿）→ published（已发布）→ disabled（已停用）
```

已发布接口统一通过 `POST /api/exec/{interfaceId}` 调用。

## 三、设计模式应用

| 设计模式 | 应用位置 | 说明 |
|----------|----------|------|
| 策略模式（Strategy） | `FieldProcessStrategy` 字段加工引擎 | Trim / Pad / Substring / TypeCast 等策略可自由扩展 |
| 建造者模式（Builder） | `QueryBuilder` / `InsertBuilder` 等 | config_json → 动态 SQL，链式构造 |
| 模板方法（Template Method） | 各 FormatConverter | 统一解析流程，子类实现具体格式 |
| 观察者/事件（Event） | `ApplicationEvent` 热更新 | sys_config 变更后广播，各模块自动刷新缓存 TTL |
| AOP 切面 | 审计 / 日志 / 性能统计 | 横切关注点与业务代码解耦 |

## 四、安全规范

| 场景 | 算法 | 存储位置 |
|------|------|----------|
| 用户登录密码 | BCrypt（不可逆） | `sys_user.password` |
| 数据源连接密码 | AES-128（可解密） | `db_connection.password` |
| 接口权限 | Sa-Token 角色白名单 | `MenuPermission` 配置 |
| 危险操作防护 | 前端 el-popconfirm 二次确认 | 删除 / 禁用等操作 |

## 六、可在面试中展开的技术亮点

1. **低代码引擎设计**：JSON 配置持久化 + 运行时 Builder 动态生成 SQL，实现零硬编码接口开发
2. **三库分离与动态多数据源**：配置库 / 业务库 / 审计库职责隔离，`dynamic-datasource` 按 dbId 切换
3. **双层缓存架构**：Caffeine 本地缓存 + Redis 分布式缓存 + SET NX PX 分布式锁，系统性解决缓存击穿
4. **AOP 异步审计**：LinkedBlockingQueue + @Async 解耦审计链路，主业务不阻塞
5. **策略模式字段加工引擎**：可扩展的字段处理策略，配置驱动，无需改代码新增处理规则
6. **多格式报文互转**：JSON / XML / CSV 统一解析为 Map 中间态，任意格式互转
