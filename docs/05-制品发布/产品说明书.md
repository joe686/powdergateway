## 1. 交付形态总览

PowerGateway 提供 **3 大类 · 7 个 SKU**，覆盖从"零依赖试用"到"生产集群"的全场景：

| 类型 | SKU 命名 | 平台 | 系统 DB | 缓存 | JDK | 前后端 | 对外发行 |
|------|---------|------|--------|------|-----|-------|--------|
| **便携版 · H2** | `powergateway-portable-win-x64-{ver}.zip` | Windows x64 | H2（内嵌） | Caffeine-only | 内嵌 JRE 17 | fat-jar 合体 | ✅ |
| **便携版 · H2** | `powergateway-portable-linux-x64-{ver}.zip` | Linux x64 | H2（内嵌） | Caffeine-only | 内嵌 JRE 17 | fat-jar 合体 | ✅ |
| **便携版 · SQLite** | `powergateway-portable-win-x64-sqlite-{ver}.zip` | Windows x64 | SQLite | Caffeine-only | 内嵌 JRE 17 | fat-jar 合体 | ✅（v1.1） |
| **便携版 · SQLite** | `powergateway-portable-linux-x64-sqlite-{ver}.zip` | Linux x64 | SQLite | Caffeine-only | 内嵌 JRE 17 | fat-jar 合体 | ✅（v1.1） |
| **标准版** | `powergateway-standard-{ver}.zip` | Win / Linux 通用 | MySQL / Oracle / OceanBase | Redis 可选（Caffeine 兜底） | 客户自备 JDK 8+ | Nginx + jar 分离 | ✅ |
| **内部测试版 · 便携** | `INTERNAL-powergateway-portable-{platform}-with-testkit-{ver}.zip` | Win / Linux | H2 + 样例业务库 | Caffeine-only | 内嵌 JRE 17 | 合体 + pg-testkit | ❌ 仅内部 |
| **内部测试版 · 标准** | `INTERNAL-powergateway-standard-with-testkit-{ver}.zip` | Win / Linux | 同标准版 + 样例业务库 | 同标准版 | 客户自备 | 同标准版 + pg-testkit | ❌ 仅内部 |

**版本命名**：`v{主}.{次}.{补丁}[-变体]`（如 `v0.1.0` / `v0.1.0-sqlite` / `v0.1.0-testkit`）
**校验方式**：每个 zip 附 `.sha256` 文件；下载后 `sha256sum` 比对
**打包体积**（v0.1.0 实测）：
- 便携版 zip **~78 MB**（含 fat-jar + 前端 + 启动脚本；未内嵌 JRE 时；内嵌 JRE 17 后 ~118 MB）
- 标准版 zip **~77 MB**（含 fat-jar + 前端 dist + 配置模板 + Nginx conf 示例）

## 2. 便携版详细规格（Portable）

### 2.1 定位
"解压即用"零依赖单机试用/演示形态。目标用户：
- 内部开发人员本地跑一个环境
- 客户 POC 快速演示
- 培训/教学场景
- 无 MySQL/Redis 的极简环境

### 2.2 系统资源要求

| 项 | 最低 | 推荐 |
|----|------|------|
| **CPU** | 2 core | 4 core |
| **内存** | 1 GB（JVM `-Xmx1g` 默认） | 2 GB |
| **磁盘** | 500 MB（zip 78MB 解压 + H2 数据初始化） | 5 GB（业务数据增长） |
| **JDK** | 内嵌 JRE 17；系统无 Java 也能跑 | — |
| **网络** | 仅本机 8080 端口 | — |

### 2.3 目录结构

```
powergateway-portable-win-x64-0.1.0/
├── backend/
│   ├── powergateway.jar         # fat-jar（前后端合体 + H2 + 全部业务代码）
│   └── lib/                     # 客户可放额外 JDBC 驱动（Oracle/OceanBase）
├── data/
│   ├── h2/                      # H2 数据文件（首次启动自动创建 pg_config.db + pg_audit.db）
│   └── logs/                    # 日志目录（默认走 stdout）
├── jre/                         # 内嵌 JRE 17（若打包时 jlink 可用；否则本目录不存在，靠系统 Java）
├── start.bat / start.sh         # 一键启动
├── stop.bat / stop.sh
├── open-console.bat             # 打开 H2 Console（http://localhost:8080/h2-console）
└── README.txt
```

### 2.4 内置能力

| 能力 | 状态 |
|------|------|
| 报文格式转换（JSON/XML/CSV/FormData） | ✅ 完整 |
| 可视化接口开发（查询/新增/修改/删除） | ✅ 完整 |
| 双层缓存 Caffeine-only（Redis 层自动降级） | ✅ |
| 系统管理（用户/日志/性能/配置） | ✅ 完整 |
| 接口转换向导 + 九步接口配置向导 | ✅ |
| 首页概览 + 报文调试 | ✅ |
| 字段公式 + 字段映射/加工 | ✅ |
| 配置导入导出（Zip + Excel + Markdown） | ✅ |
| H2 Console 数据管理 | ✅（默认开，仅本机访问） |
| Swagger 3.0 API 文档 | ✅ |

### 2.5 功能限制（相比标准版）

| 功能 | 说明 |
|------|------|
| ❌ **分库分表** | H2 单库限制，无法演示分片路由；如需 v1.1 SQLite 分支同样受限 |
| ❌ **Redis 分布式缓存** | 默认强制 Caffeine-only；多实例部署无缓存共享 |
| ❌ **注册中心（Nacos/Eureka）** | provided scope 未打包，无 `service://` 协议能力；如需，改用 `-Pregistry-nacos` 或 `-Pregistry-full` 重打包 |
| ⚠️ **多用户并发** | H2 单进程限制，建议 ≤10 并发用户；生产环境用标准版 |
| ⚠️ **数据持久性** | H2 文件模式安全，但无主从复制/热备；重要数据请定期备份 `data/h2/` 目录 |
| ⚠️ **性能上限** | 建议 QPS < 200；高并发用标准版 + MySQL |

### 2.6 启动流程

```bash
# Windows
双击 start.bat

# Linux
chmod +x start.sh && ./start.sh
```

- 首次启动 **~15 秒**（初始化 H2 + 灌 init.sql + Spring Boot 启动）
- 浏览器打开 http://localhost:8080
- 默认账号 `admin / Admin@123`

### 2.7 数据存储

- 配置库：`data/h2/pg_config.mv.db`（H2 单文件）
- 审计库：`data/h2/pg_audit.mv.db`
- 备份 = 复制 `data/h2/` 目录
- H2 Console 连接串：`jdbc:h2:file:./data/h2/pg_config`（用户 `sa` / 密码空）

## 3. 标准版详细规格（Standard）

### 3.1 定位
生产环境正式部署形态：前后端分离 · 客户自建 DB · 支持多实例集群。目标用户：
- 客户正式生产环境
- 需要多实例负载均衡
- 与客户现有 MySQL / Redis / Nginx 集成
- 走 CI/CD 自动化部署

### 3.2 系统资源要求

| 项 | 最低（单实例） | 推荐（生产） |
|----|-------------|-------------|
| **CPU** | 4 core | 8 core / 实例 |
| **内存** | 2 GB（JVM `-Xmx2g` 默认） | 4 GB / 实例 |
| **磁盘（应用）** | 200 MB | 5 GB |
| **磁盘（数据库）** | 视业务规模 | 独立存储卷 |
| **JDK** | JDK 8+ | JDK 17 LTS |
| **网络** | 8080 后端 + 80/443 前端 Nginx | 内网隔离 + 反代 |

### 3.3 目录结构

```
powergateway-standard-0.1.0/
├── backend/
│   ├── powergateway-backend.jar          # 后端 fat-jar（不含前端）
│   ├── config/
│   │   ├── application-prod.yml.example  # 配置模板（需 cp 为 .yml 并填写）
│   │   └── application-prod.yml          # ← 客户创建
│   ├── lib/
│   │   └── README-drivers.txt            # 客户放 Oracle/OceanBase/达梦驱动
│   ├── scripts/
│   │   ├── install-schema-mysql.sh       # 建库建表脚本
│   │   ├── start.sh / stop.sh / start.bat
│   ├── init-sql/
│   │   ├── init-mysql.sql                # 配置库 DDL
│   │   └── init-audit-mysql.sql          # 审计库 DDL
│   └── logs/                             # 运行日志（start.sh 自动 nohup）
├── frontend/
│   ├── dist/                             # 前端静态资源（部署到 Nginx）
│   └── nginx.conf.example                # Nginx 反代模板
├── docs/                                 # 保留目录（v1.1 补部署手册 PDF）
└── README.md
```

### 3.4 内置能力

同便携版全部能力 **+**：

| 能力 | 说明 |
|------|------|
| ✅ **Redis 双层缓存** | `pg.cache.redis.enabled=true` 时生效；`false` 时降级 Caffeine-only |
| ✅ **多实例集群** | 无状态设计，前置负载均衡即可水平扩展 |
| ✅ **分库分表** | MySQL/Oracle 环境下支持 `ShardRouter`（取模/范围/补查） |
| ✅ **审计库独立** | 审计日志通过 `@DS("audit")` 写独立库，可用不同 MySQL 实例 |
| ✅ **注册中心（可选）** | 打包时 `-Pregistry-nacos` 或 `-Pregistry-full` 激活；默认包不带 |
| ✅ **接口热发布** | `POST /api/exec/{id}` 统一执行入口，配置变更实时生效 |

### 3.5 启动流程

```bash
# 1. 建库（在部署机上执行一次）
cd backend/scripts
./install-schema-mysql.sh <MYSQL_HOST> <PORT> <USER> <PASS> [config_db] [audit_db]
# 默认库名 powergateway_config / powergateway_audit

# 2. 配置连接
cd backend/config
cp application-prod.yml.example application-prod.yml
vim application-prod.yml     # 改数据库 URL/账号/密码 + Redis + AES key

# 3. 启动
./scripts/start.sh
tail -f logs/powergateway.log

# 4. 部署前端到 Nginx
sudo cp -r frontend/dist /opt/powergateway/frontend/
sudo cp frontend/nginx.conf.example /etc/nginx/sites-available/powergateway
sudo ln -s /etc/nginx/sites-available/powergateway /etc/nginx/sites-enabled/
sudo systemctl reload nginx
```

- 后端启动 **~9 秒**（连接 MySQL + 初始化 Spring Boot 上下文）
- 前端访问 http://<你的域名>/
- 默认账号 `admin / Admin@123`

### 3.6 高可用部署建议

- 后端：≥ 2 实例 + Nginx `upstream` 负载均衡 + `proxy_next_upstream` 故障切换
- MySQL：主从复制 或云 RDS 高可用
- Redis：Sentinel 或 Cluster 模式
- 前端：Nginx 直接托管 `frontend/dist/`，可开 CDN 加速

### 5.2 业务数据库（PG 转接口的目标系统 DB）

支持通过「数据库连接管理」页面动态添加，格式：`{db_type}:{host}:{port}/{schema}`

| 类型 | 支持状态 | 驱动 |
|------|--------|------|
| MySQL | ✅ | 内置 |
| Oracle | ✅ | 内置 |
| PostgreSQL | ✅ | 内置 |
| OceanBase | ✅ | 客户放 lib/ |
| 其他 JDBC 兼容库 | ⚠️ 客户自扩展 | 客户放 lib/ + `application-prod.yml` 声明 driver-class-name |

### 5.3 驱动扩展方式

标准版部署后添加新驱动：
1. 到官方渠道下载 JDBC jar（如 ojdbc11-21.9.0.0.jar）
2. 复制到 `backend/lib/` 目录
3. 修改 `application-prod.yml` 里对应连接的 `driver-class-name`
4. 重启后端服务

启动脚本已用 `-Dloader.path="$BASE/lib"` 让 fat-jar 加载 lib/ 目录的额外 jar。

### 6.1 注册中心裁剪原理

- 默认 pom.xml 里 `nacos-client` 和 `eureka-client` 都是 `provided` scope
- `spring-boot-maven-plugin` 配置 `<excludeGroupIds>com.alibaba.nacos,com.netflix.eureka</excludeGroupIds>` 精准排除
- 打包时用 `-Pregistry-nacos` 或 `-Pregistry-full` 激活对应 profile，其中的 `<dependency>` 以 `compile` scope 覆盖默认 `provided`
- 校验方式：
  ```bash
  mvn dependency:list -DincludeScope=runtime                # 默认无 nacos/eureka
  mvn dependency:list -DincludeScope=runtime -Pregistry-full # 包含全部
  ```

mysqldump -u root -p powergateway_audit  > audit_$(date +%Y%m%d).sql

## 11. 常见问题

### Q1: 便携版启动报"端口 8080 被占用"？
**A**: `application-standalone.yml` 里改 `server.port` 或用命令行覆盖：
```bash
java -jar backend/powergateway.jar --server.profiles.active=standalone --server.port=8090
```

### Q2: 标准版启动报"Config data resource ... does not exist"？
**A**: 未复制 `application-prod.yml.example` → `application-prod.yml`。start.sh 会检测并提示。

### Q3: 便携版可以给客户看/内网访问吗？
**A**: 默认 H2 Console 只允许本机（`web-allow-others: false`），API 无 IP 限制。如需公网访问，建议：改配置固定管理员密码、关闭 h2-console、走反代加 HTTPS。

### Q4: Redis 一定要装吗？
**A**: 不。便携版强制关闭；标准版 `pg.cache.redis.enabled: false` 即降级 Caffeine-only。**代价**：多实例集群时无缓存共享，每个实例独立缓存；单实例部署无影响。

### Q5: Oracle 驱动版权问题？
**A**: Oracle 官方 ojdbc8 驱动可以二次分发（在 OTN License 允许范围内）；ojdbc11 有更严格限制建议客户从 Oracle 官网下载。达梦 / 金仓 / OceanBase 等国产库驱动版权归各自厂商，请客户自行获取。

### Q6: 便携版能改成 MySQL 后端吗？
**A**: 能，但不推荐（那就是标准版的场景）。方法：编辑 `backend/config/application-standalone.yml` 里的 `spring.datasource.dynamic.datasource.master.url` 换成 MySQL 连接串 + 保留 `pg.cache.redis.enabled: false`；重启即可。生产还是建议用标准版。

### Q7: 我要修改数据库字段/加一列，怎么办？
**A**: 阶段七以内不建议客户直接改表结构（会破坏 MyBatis-Plus 实体映射）。请提 Issue，我们出正式迁移脚本 + entity 更新版本。
