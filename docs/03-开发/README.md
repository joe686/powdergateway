# 03-开发

开发过程记录目录。三类内容：**整体排期 + 范围变更 + Bug 追踪**，外加每个单元的实现计划。

## 总览文档

| 文件 | 用途 | 维护节奏 |
|------|------|---------|
| [开发计划.md](./开发计划.md) | 五阶段 28 个单元的整体路线图与进度 | 单元交付后更新 |
| [变更记录.md](./变更记录.md) | 已交付单元的范围增减（CHG-XXX 编号） | 每次范围变更后追加 |
| [问题清单.md](./问题清单.md) | Bug 与待办（BUG-XXX 编号） | 发现即录入，修复后标记 |

## 单元任务计划

[任务计划/](./任务计划/) 下每份对应一个交付单元的实现拆分（步骤、Red/Green/Refactor 节奏、依赖）。命名同设计文档：

```
YYYY-MM-DD-{单元编号}-{特征词}.md
例：2026-05-13-sys3-user-permission.md
```

## 范围变更规约

**任何对已交付单元的功能增减必须三步同步**，详见 [变更记录.md](./变更记录.md) 头部说明：

1. 在 `变更记录.md` 追加一条 `CHG-XXX`
2. 更新 [../01-需求/需求拆分与最小实现方案.md](../01-需求/需求拆分与最小实现方案.md)
3. 更新本目录的 [开发计划.md](./开发计划.md)

> Bug 修复若不改变对外行为，**不需要**走 CHG 流程。
