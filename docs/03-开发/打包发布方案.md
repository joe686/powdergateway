## 一、需要回答的三个问题

| 问题 | 选项 | 影响 |
|------|------|------|
| A · **目标平台** | Windows / Linux / 两者 | 打包格式、启动脚本 |
| B · **数据库形态** | 外置 MySQL / 内嵌 H2 / 内嵌 SQLite / 内嵌 MySQL | 部署难度、性能上限 |
| C · **中间件形态** | 外置 Redis / 内嵌 Caffeine / 按环境自动切 | 依赖复杂度、性能 |

## 二、中间件裁剪矩阵

| 中间件 | 生产（Linux · 大规模） | 试用（Win/Linux · 单机） | 演示（笔记本） |
|--------|----------------------|------------------------|---------------|
| **MySQL 配置库** | 外置 MySQL 8 | 外置 MySQL 8 | **内嵌 H2**（文件模式） |
| **MySQL 审计库** | 外置 MySQL 8（独立实例） | 外置 MySQL 8（同实例） | **内嵌 H2** |
| **业务库**（多数据源，用户配置） | 外置（用户自己的） | 外置（用户自己的） | pg-testkit 样例 H2 |
| **Redis 缓存** | 外置 Redis 6 | 外置 Redis 6 | **仅 Caffeine 本地缓存** |
| **注册中心**（Nacos/Eureka） | 外置（选装） | 外置（选装） | 不装 |

**代码侧支撑**：Spring Boot profile 三档：
- `prod` → 全外置
- `standalone` → MySQL 外置 + Redis 可选（自动降级到 Caffeine-only）
- `demo` → 全内嵌 H2/Caffeine

Redis 自动降级已经有基础（backend 有 Caffeine + Redis 双层，仅去掉 Redis 层不改代码逻辑），只需要一个开关。

## 三、四套发布方案（供用户挑）

### 方案 1 · **便携 zip（Portable）** 🌟推荐给试用/演示场景

**内容**：
```
powergateway-portable-{version}-{platform}.zip
├── backend/
│   ├── powergateway.jar               # fat-jar，含前端静态资源
│   ├── config/
│   │   └── application-standalone.yml
│   └── logs/
├── data/
│   ├── h2/                            # H2 数据文件（首次启动创建）
│   └── caffeine-only/                 # 无 Redis 标志
├── jre/                               # 内嵌 JRE 17（~40MB）
├── start.bat                          # Win 一键启动
├── start.sh                           # Linux 一键启动
├── stop.bat / stop.sh
└── README.txt                         # 端口占用/账号密码/常见问题
```

**特点**：
- 解压即用，**无需装 Java、MySQL、Redis**
- 单机部署，H2 存所有配置（DB 文件在 `data/h2/`）
- 前后端一体，浏览器打开 http://localhost:8080 即用
- 体积 ~150MB（含 JRE + 依赖）

**限制**：
- H2 并发和性能有限，适合单人试用
- 无审计库分离
- 重启数据保留（H2 文件持久化）

**平台**：Win / Linux 两个包分别构建

---

### 方案 2 · **标准部署包（Standard）** 🌟推荐给客户正式部署

**内容**：
```
powergateway-{version}.zip
├── powergateway.jar
├── config/
│   ├── application.yml
│   ├── application-prod.yml.example    # 模板，需用户填
│   └── logback-spring.xml
├── scripts/
│   ├── install-mysql-schema.sh/.bat    # 建库建表脚本
│   ├── start.sh / start.bat
│   ├── stop.sh / stop.bat
│   └── restart.sh / restart.bat
├── docs/
│   └── 部署手册.pdf
└── README.md
```

**前置要求**（客户自备）：
- Java 17 JDK
- MySQL 8.x（配置库 + 审计库，可同一实例）
- Redis 6.x（可选，无则自动降级 Caffeine-only）

**特点**：
- 轻量（~80MB，无 JRE）
- 生产可用（MySQL + Redis）
- 有部署手册和运维脚本

**平台**：Win / Linux 通用（jar + 脚本适配）

---

### 方案 3 · **Docker 一键（Compose）** 🌟推荐给容器化客户

**内容**：
```
powergateway-docker-{version}/
├── docker-compose.yml
├── .env.example
├── powergateway.jar
├── Dockerfile
└── init/
    └── init.sql               # MySQL 首次启动灌 schema
```

`docker-compose.yml` 起 4 个容器：
- powergateway（backend + 前端）
- mysql:8
- redis:6
- pg-testkit（选装）

**特点**：
- 客户只要装 Docker，`docker compose up -d` 一键起
- 数据卷持久化 MySQL/Redis 数据
- 版本升级只换镜像 tag
- 支持 Nacos/Eureka 容器一起编排（可选 profile）

**平台**：Linux 首选，Win 需 Docker Desktop

---

### 方案 4 · **Windows 原生安装包（MSI/EXE）**

**工具**：`Launch4j`（jar → exe）+ `Inno Setup`（打成 MSI）

**特点**：
- Win 用户双击安装，注册为 Windows 服务
- 开机自启，右下角托盘图标
- 卸载走控制面板

**代价**：
- 增加打包脚本复杂度
- 只支持 Windows
- 需要签名证书（自签名会有 SmartScreen 警告）

**平台**：Win 独有

按你（用户）当前"马上要发布做测试"的诉求，**第一版发布**建议：

## 六、便携版（方案 1）技术细节

### 6.1 内嵌 H2 的具体做法

在 `application-standalone.yml`：
```yaml
spring:
  datasource:
    dynamic:
      primary: master
      datasource:
        master:                          # 配置库
          url: jdbc:h2:file:./data/h2/pg_config;MODE=MySQL;DB_CLOSE_DELAY=-1;AUTO_SERVER=TRUE
          driver-class-name: org.h2.Driver
          username: sa
          password:
        audit:                           # 审计库
          url: jdbc:h2:file:./data/h2/pg_audit;MODE=MySQL;DB_CLOSE_DELAY=-1
  h2:
    console:
      enabled: true                      # 内置管理界面（方便运维）
      path: /h2-console
```

**兼容性**：现有代码已用 `dynamic-datasource`，H2 MySQL 模式基本兼容 MySQL 8 语法。已知需要处理的差异：
- MySQL `DATE_FORMAT` → 已有 `H2DateFormatAlias`
- MySQL `IFNULL` → H2 用 `NVL`（可加 alias）
- `LIMIT ? OFFSET ?` → 通用

### 6.2 无 Redis 降级方案

代码侧已支持双层缓存（Caffeine + Redis）。加一个 profile：
```yaml
# application-caffeine-only.yml
pg:
  cache:
    redis:
      enabled: false                    # 关掉 Redis 层
```

对应代码：`CacheConfig.java` 里 `@ConditionalOnProperty` 控制 Redis bean 注入。**已有基础，需要小改**（0.5 天）。

### 6.3 内嵌 JRE

用 `jlink` 从 JDK 17 裁剪最小 JRE（约 40MB）：
```bash
jlink --module-path $JAVA_HOME/jmods --add-modules java.base,java.sql,java.desktop,java.management,jdk.unsupported,... --output jre --strip-debug --compress=2 --no-header-files --no-man-pages
```

Win / Linux 分别 jlink 一次。

### 6.4 前后端合体

Spring Boot 内嵌前端静态资源到 `src/main/resources/static/`：
- 构建流程：`npm run build` → 复制 `frontend/dist/*` 到 `backend/src/main/resources/static/`
- Maven `resources` 插件配置好，`mvn package` 一步出 fat-jar

已有 `frontend/dist/`（现在也在 commit 里，可以直接用）。

### 6.5 启动脚本 `start.bat`

```bat
@echo off
chcp 65001
title PowerGateway

set JAVA=%~dp0jre\bin\java.exe
if not exist "%JAVA%" (
  echo 未找到内嵌 JRE，使用系统 Java
  set JAVA=java
)

echo 正在启动 PowerGateway...
echo 数据目录: %~dp0data
echo 访问地址: http://localhost:8080
echo 默认账号: admin / Admin@123
echo.
echo 关闭本窗口即停止服务。
echo.

"%JAVA%" -Xms256m -Xmx1g -Dfile.encoding=UTF-8 -Dspring.profiles.active=standalone -jar "%~dp0backend\powergateway.jar"
pause
```

**注意**：`chcp 65001` 处理中文乱码（这个坑你在 memory 里已经记过）。

### 6.6 pg-testkit 是否打包

- 便携版默认**不带** pg-testkit（体积考虑）
- 提供 `powergateway-portable-{platform}-with-testkit.zip` 版本，供需要样例库/Mock 后端的客户

## 七、注册中心依赖裁剪（配合 REG-1 方案）

Maven profile 三档：
```xml
<profiles>
  <profile>
    <id>registry-none</id>
    <activation><activeByDefault>true</activeByDefault></activation>
    <!-- 不带 nacos-client / eureka-client -->
  </profile>
  <profile>
    <id>registry-nacos</id>
    <dependencies>
      <dependency>...nacos-client...</dependency>
    </dependencies>
  </profile>
  <profile>
    <id>registry-full</id>
    <dependencies>
      <dependency>...nacos-client...</dependency>
      <dependency>...eureka-client...</dependency>
    </dependencies>
  </profile>
</profiles>
```

打包命令：
```bash
mvn clean package -Pregistry-nacos        # 出 nacos 版
mvn clean package -Pregistry-full         # 出全功能版
mvn clean package                          # 出无注册中心版（最小）
```

对应发布 SKU：
- `powergateway-portable-{platform}.zip` — 无注册中心
- `powergateway-portable-{platform}-nacos.zip` — 带 Nacos
- `powergateway-portable-{platform}-full.zip` — Nacos + Eureka

## 八、发布流水线（v1 手工版，v2 CI）

### 8.1 v1 手工流程

```bash
# 1. 前端构建
cd frontend && npm ci && npm run build

# 2. 后端打包（选 profile）
cd backend && mvn clean package -DskipTests -Pregistry-none

# 3. 组装便携包
bash scripts/build-portable.sh --platform win     # 或 linux

# 4. 产物在 dist/ 下
ls dist/powergateway-portable-win-0.1.0.zip
```

需要写：
- `scripts/build-portable.sh` （组装脚本）
- `scripts/jlink-jre.sh` （裁剪 JRE）

### 8.2 v2 GitHub Actions（若代码托管在 GitHub）

一次 push tag，自动跑 win/linux 两个 job，出两个 zip 挂到 Release。
