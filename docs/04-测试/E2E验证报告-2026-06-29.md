### 12.2 本轮新学到的入参约定（已知差异，非 BUG）

| 场景 | 错误约定 | 正确约定 |
|---|---|---|
| DELETE configJson | `{"tables":[{...}], "conditions":[...]}`（与 UPDATE 同级）| `{"tables":[{"tableName":"...","conditions":[...]}]}` — conditions 嵌入 table 内 |
| INSERT 字段定义 | `{name, sourceKey, sourceValue}` | `{tableName, paramKey, constValue}` — 与 SELECT 的 `{name, alias}` 不一致 |
| Template save | `mappingRule`（单数）| `mappingRules`（复数；持久化时序列化为 `mapping_rule` 单数）|
| Template save | `processRule` 字段 | 暂未在 `TemplateSaveRequest` 中提供，需直接 SQL UPDATE convert_template.process_rule（已在 M16ConvertApiTest 中记录为已知限制）|
| ExecRequest | `{key:val}` 直接 | `{params:{key:val}}` 嵌套 |
| `/api/cache/{id}/...` | `/api/interface/{id}/cache/...` | 正确路径在 `/api/cache/` 域 |

> 这些可作为 **OBS-8 统一接口契约** 的输入：建议将 INSERT/UPDATE/DELETE 三处的字段命名差异（tableName vs name、paramKey vs sourceKey、constValue vs sourceValue）在不破坏现有合同的前提下逐步收敛。当前可加文档明示。
