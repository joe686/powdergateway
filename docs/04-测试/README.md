## TDD 强制约束（摘要）

详见 [TDD规范.md](./TDD规范.md)：

1. **Red** — 先写测试，运行确认失败
2. **Green** — 写最小实现使测试通过
3. **Refactor** — 测试全绿后重构并复跑

测试类命名：`{单元编号}{功能}Test.java`，**必须加** `@ActiveProfiles("test")`（否则会连生产 MySQL/Redis）。

## 测试工具模块

`pg-testkit/` 是独立的测试工具模块（端口 8081，内嵌 Mock 服务器 9999），通过 HTTP/JDBC 与 backend 集成，不互相 import 内部类。详见 [测试工具计划.md](./测试工具计划.md) 与 [`pg-testkit/README.md`](../../pg-testkit/README.md)。
