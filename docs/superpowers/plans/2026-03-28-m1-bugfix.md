## Task 1：修复 F-4 — fixedValue 空字符串被吃掉

**Files:**
- Modify: `frontend/src/views/convert/FieldMapping.vue:524`

**问题代码（第 524 行）：**
```js
fixedValue: r.fixedValue || null
```
`||` 把 `""` 视为 falsy，`""` → `null`，导致空字符串固定值保存后回填丢失。

- [ ] **Step 1：定位代码**

打开 `frontend/src/views/convert/FieldMapping.vue`，找到第 515-525 行的模板回填逻辑：

```js
mappingRules.value = rawRules.map(r => ({
  id: ++ruleIdCounter,
  targetField: r.targetField || '',
  srcField: (r.fixedValue !== null && r.fixedValue !== undefined) ? null : (r.srcField || null),
  useFixed: r.fixedValue !== null && r.fixedValue !== undefined,
  fixedValue: r.fixedValue || null   // ← 第 524 行，这里有 bug
}))
```

- [ ] **Step 2：修改第 524 行**

将：
```js
fixedValue: r.fixedValue || null
```
改为：
```js
fixedValue: r.fixedValue ?? null
```

`??` 只在 `null` 或 `undefined` 时取右侧默认值，空字符串 `""` 会被正确保留。

- [ ] **Step 3：手动验收**

1. 启动后端（`mvn spring-boot:run`）和前端（`npm run dev`）
2. 进入"字段映射配置"页面，新建或编辑一条模板
3. 为某个映射规则勾选"固定值"，在输入框中输入空字符串（清空后保存）
4. 保存模板，重新打开该模板的编辑弹窗
5. 预期：勾选框仍为"固定值"，输入框显示空字符串（而非切换为源字段模式）

## Task 2：修复 F-7 — JSON 解析异常静默

**Files:**
- Modify: `frontend/src/views/convert/FieldMapping.vue:517`

**问题代码（第 517 行）：**
```js
try { rawRules = JSON.parse(tpl.mappingRule) } catch (_) {}
```
JSON 解析失败时 `rawRules` 维持 `[]`，规则区静默清空，用户无法得知模板数据损坏。

- [ ] **Step 1：修改第 517 行**

将：
```js
try { rawRules = JSON.parse(tpl.mappingRule) } catch (_) {}
```
改为：
```js
try {
  rawRules = JSON.parse(tpl.mappingRule)
} catch (_) {
  ElMessage.warning('模板映射规则解析失败，已清空规则，请重新配置')
}
```

确认文件顶部已有 `import { ElMessage } from 'element-plus'`（或通过全局自动导入，当前项目已配置 auto-import，直接使用即可）。

- [ ] **Step 2：手动验收**

1. 直接修改数据库中某条模板的 `mapping_rule` 字段为非法 JSON（如 `"{broken"`）
2. 在字段映射页面加载该模板
3. 预期：页面弹出 warning toast "模板映射规则解析失败，已清空规则，请重新配置"，规则区为空
