# SYS-1 日志管理 设计文档

**日期**：2026-05-13  
**单元**：SYS-1  
**状态**：已批准

---

## 1. 背景与目标

SYS-1 在已完成的 M2-9 SQL 审计日志基础上，补充**系统操作日志**（`sys_log`）的写入和查询能力，并提供前端日志管理页面。核心价值：运维人员可按条件搜索所有写操作记录，导出 Excel 留档；管理员可通过 `sys_config` 开关控制日志菜单的可见性。

---

## 2. 范围

**包含：**
- `sys_log` 表建表 DDL + `SysLog` 实体
- `@SysLog` 注解 + `SysLogAspect` AOP 切面（拦截写操作 Controller 方法）
- `SysLogService`（异步队列写入，与 M2-9 模式一致）
- `GET /api/log/list`（分页 + 多条件过滤）
- `GET /api/log/export`（Apache POI 导出 Excel）
- `GET /api/log/audit/list`（SQL 审计日志分页查询，新增接口复用 sql_audit_log 表）
- 前端 `views/system/LogList.vue`（双 Tab：操作日志 + SQL 审计）
- `sys_config` 补充 `log_menu_enabled=true` 默认值
- 测试：4 个测试类，覆盖 AOP 逻辑、Service 写入/查询、Controller 接口

**不包含：**
- 日志打印本身（Logback 配置在 P0-1 完成）
- M2-9 `sql_audit_log` 的写入逻辑（已完成）
- SQL 审计日志的 Excel 导出（不在本单元范围）
- 日志留存清理定时任务（复用 `AuditLogCleanupJob` 中的 `sql.log.retention.days` 配置即可，`sys_log` 清理待 SYS-4 统一配置）

## 3. 数据库 Schema

### 3.1 sys_log 表（新增，配置库）

```sql
CREATE TABLE IF NOT EXISTS sys_log (
  id        BIGINT PRIMARY KEY AUTO_INCREMENT,
  module    VARCHAR(64)  COMMENT '操作模块（如"模板管理"）',
  action    VARCHAR(128) COMMENT '操作动作（如"保存模板"）',
  operator  VARCHAR(64)  COMMENT '操作人（Sa-Token loginId，未登录时为 system）',
  op_ip     VARCHAR(64),
  op_time   DATETIME     DEFAULT CURRENT_TIMESTAMP,
  level     VARCHAR(16)  COMMENT 'INFO / ERROR',
  error_msg TEXT         COMMENT '失败时的错误信息（ERROR 级别）',
  cost_ms   INT          COMMENT '执行耗时（ms）'
);
```

### 3.2 sys_config 补充默认值

```sql
INSERT IGNORE INTO sys_config (config_key, config_value, description)
VALUES ('log_menu_enabled', 'true', '日志管理菜单显示开关（true/false）');
```

> SYS-3 的 `MenuPermission.LOG_MENU_CONFIG_KEY = "log_menu_enabled"` 已引用此 key，补充默认值后开关逻辑自动生效。

## 4. 后端设计

### 4.1 新增文件清单

| 文件 | 说明 |
|------|------|
| `model/SysLog.java` | 实体类，`@TableName("sys_log")`，无软删除字段 |
| `dao/SysLogMapper.java` | MyBatis-Plus Mapper |
| `aop/SysLog.java` | 注解，参数：`String module`、`String action` |
| `aop/SysLogAspect.java` | AOP 切面，`@Around("@annotation(com.powergateway.aop.SysLog)")` |
| `service/SysLogService.java` | 异步队列写入 + 分页查询 + Excel 导出 |
| `controller/SysLogController.java` | REST 层：list / export / audit/list |

### 4.2 @SysLog 注解定义

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface SysLog {
    String module();
    String action();
}
```

### 4.3 SysLogAspect 核心逻辑

```
1. 记录 startMs = System.currentTimeMillis()
2. 从 Sa-Token 获取 operator，从 RequestContextHolder 获取 op_ip
3. 执行目标方法
4. 成功 → level=INFO，cost_ms = now - startMs，投入队列
5. 抛异常 → level=ERROR，error_msg = truncate(e.getMessage(), 1000)，投入队列，re-throw
```

### 4.4 SysLogService 写入机制

与 M2-9 `AuditLogService` 相同模式：
- `LinkedBlockingQueue<SysLog>`，容量 10000
- `@PostConstruct` 启动单个守护线程消费队列，逐条 `sysLogMapper.insert(log)`
- 队列满时静默丢弃（`offer` 非阻塞）

### 4.6 API 接口

**操作日志列表：**
```
GET /api/log/list
参数：operator, module, level, startTime, endTime, page, size
返回：Result<IPage<SysLog>>
```

**操作日志导出：**
```
GET /api/log/export
参数：同 list（无分页）
返回：application/octet-stream（Excel 文件）
```

**SQL审计日志列表（新增）：**
```
GET /api/log/audit/list
参数：opType(INSERT/UPDATE/DELETE), result(SUCCESS/FAIL), startTime, endTime, page, size
返回：Result<IPage<SqlAuditLog>>
```

## 7. 验收标准

1. 执行一次"保存模板"操作，`sys_log` 表有一条 `level=INFO`、`module=模板管理`、`action=保存模板` 的记录
2. 执行失败的写操作（如保存非法数据），`sys_log` 有对应 `level=ERROR` 记录
3. 前端操作日志 Tab 按条件过滤后结果正确
4. 点击"导出 Excel"下载文件，包含当前过滤结果
5. SQL 审计 Tab 展示 sql_audit_log 数据，空 operator/ip 显示 `—`
6. `sys_config` 中将 `log_menu_enabled` 设为 `false` 后，重新调用 `/api/auth/menu`，返回列表中不含 `/system/log`
