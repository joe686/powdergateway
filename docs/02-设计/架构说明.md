## 两大核心模块

### 模块一：报文格式转换

面向接口报文的格式互转与字段加工，配置结果保存为可复用模板。

- **格式转换**：JSON ↔ XML ↔ CSV ↔ FormData（可扩展 SOAP、Protobuf）
- **字段映射**：拖拽方式配置源字段 → 目标字段的对应关系
- **字段处理**：截取、补位、类型转换、四则运算等
- **渠道模板**：识别渠道标识字段，按渠道应用不同转换模板
- **模板管理**：模板的保存、复用、编辑、删除

```
请求报文 → 格式解析 → 字段映射 → 字段处理 → 渠道模板匹配 → 输出报文
                                                      ↓
                                               模板持久化（配置库）
```

### 模块二：可视化接口开发（CRUD）

通过界面配置生成 SQL 并对接目标数据库，支持高并发访问。

- **数据库连接**：可视化添加 MySQL/Oracle/PostgreSQL 数据源
- **CRUD 配置**：
  - 查询：多表关联、字段筛选、条件构建、数据加工
  - 新增：请求字段校验、自定义字段、计算字段
  - 修改：唯一条件强制、修改前后数据对比
  - 删除：安全确认、防批量误删、唯一条件校验
- **分库分表**：独立菜单配置分片规则
- **缓存策略**：双层缓存（Caffeine 本地 + Redis 分布式），支持缓存预热和防击穿
- **字段公式**：可复用的复杂查询条件和计算逻辑
- **数据审计**：所有增删改操作异步写入独立审计库（含操作人、IP、时间、SQL、执行结果）

```
可视化配置 → SQL生成 → 分库分表解析 → 数据库执行 → 字段加工 → 缓存写入 → 响应
                                              ↓（异步）
                                        审计日志 → 审计库
```

## 前端菜单结构

```
首页
  └─ 系统概览 / 新手引导
接口转换配置（模块一）
  └─ 报文格式转换 / 字段转换配置 / 字段加工配置 / 渠道模板管理 / 转换模板管理
可视化接口开发（模块二）
  └─ 数据库连接管理 / 表结构管理 / 接口开发 / 分库分表配置
  └─ 常用字段公式管理 / 缓存查询管理 / 接口配置导航（9步向导）
系统管理
  └─ 日志管理 / SQL留存管理 / 性能统计 / 用户权限管理 / 系统配置
辅助工具
  └─ 接口文档（Swagger） / 报文调试 / 插件管理
```

菜单支持角色权限控制（admin/user/readonly）和开关控制（日志菜单等）。

## 配置库核心表（9张）

| 表名 | 说明 |
|------|------|
| `sys_user` | 用户表，BCrypt 密码 |
| `convert_template` | 转换模板，含 mapping_rule、process_rule（JSON） |
| `channel_config` | 渠道配置，渠道码 + 识别字段 + 关联模板 |
| `port_route` | 端口分发路由，`channel_code`+`port_address`+`request_template_id`（A→B）+`response_template_id`（B→A） |
| `db_connection` | 数据源，密码 AES 加密，支持 MySQL/Oracle/PG |
| `interface_config` | 接口配置，config_json 存全量配置，status: draft/published/disabled |
| `shard_config` | 分库分表规则 JSON |
| `field_formula` | 可复用字段公式 |
| `sys_config` | 全局 KV 配置（缓存TTL、日志留存天数等） |

审计库单独建：`sql_audit_log`（含 before_snapshot、op_ip、sql_text）。

## 平台支撑能力（阶段七 · 2026-07-22 立项）

### 服务注册与发现（REG-1）

- **抽象层**：`RegistryClient` 接口 + `NacosRegistryClient` / `EurekaRegistryClient` 两实现 + `RegistryFacade` 门面（聚合所有已启用客户端）
- **数据表**：`registry_config`（type / server_addr / namespace / auth / register_self / service_name / extra_metadata），密码 AES 加密
- **自注册**：`ApplicationReadyEvent` → 遍历 `enabled=1 && register_self=1` 的记录 register；`ContextClosedEvent` → deregister
- **服务发现**：`port_route.port_address` 支持 `service://SVC/path` 协议，`ServiceUrlResolver` 在 `DispatchService.dispatch` 前解析为具体 URL，30s 本地缓存兜底
- **心跳告警**：连续 3 次失败 → 写 `perf_alert` alertType=`REGISTRY_HEARTBEAT_FAIL`（复用 `AlertService`）
- **Maven profile 三档**：`registry-none`（默认）/ `registry-nacos` / `registry-full`；未启用时 Facade 为 Noop 实现，零依赖
- **前端菜单**：「辅助工具 → 注册中心管理」`/tools/registry`；端口路由弹窗「选择服务」按钮

### 测试与自测环境（TEST-1 · pg-testkit 增强）

- **架构约束**：`pg-testkit` 保持独立 Spring Boot 进程（8081 端口），主 backend **零耦合**（不 import pg-testkit 内部类），遵守项目"三模块互不 import"约束
- **前端集成**：嵌入 PG 主前端独立目录 `frontend/src/modules/testkit/`，通过 **TESTER 角色**权限白名单 + Vite build 参数双控可见性；下线时删目录即可，生产环境不预置 tester 用户
- **样例业务库**：`pg_demo_biz`（+ 3 shard 库）10 张 MySQL 表，覆盖 CRUD/JOIN/分库分表/缓存演示；Faker 生成中文语义数据；幂等 init + reset + drop API
- **Mock 后端**：可视化 GUI 增删改规则 + SQLite 请求历史 + 回放，端口可配（默认 9999）

### 打包发布形态（REL-1）

平台交付 7 个 SKU，Maven profile 组合（`db-h2` / `db-sqlite` × `registry-none` / `registry-nacos` / `registry-full`）控制形态：

| 类型 | 平台 | 系统 DB | 缓存 | JDK | 前后端 | 对外 |
|------|------|--------|------|-----|-------|------|
| 便携 H2 | Win / Linux x64 | H2（MODE=MySQL） | Caffeine-only | 内嵌 JRE 17 | fat-jar | ✅ |
| 便携 SQLite | Win / Linux x64 | SQLite | Caffeine-only | 内嵌 JRE 17 | fat-jar | ✅ |
| 标准 | 通用 | MySQL / Oracle / OceanBase | Redis 可选 | 客户自备 8+ | 分离（Nginx + jar） | ✅ |
| 内部测试 | Win / Linux / 通用 | 同上 + pg-testkit | 同上 | 同上 | 同上 | ❌ 仅内部 |

- **发布流程**：主分支 + tag 触发 GitHub Actions（`v0.1.0` / `v0.1.0-sqlite` / `v0.1.0-testkit` 三种 tag 命名），每个 SKU 冒烟测试通过后自动上传 zip + SHA256 到 Release
- **便携版启动**：`start.bat` / `start.sh` 一键起，读内嵌 JRE + fat-jar，H2 数据文件在 `data/h2/`，客户浏览器打开 http://localhost:8080 即用

## 配置库核心表（10 张 · 2026-07-22 更新）

在原有 9 张表基础上，阶段七新增：

| 表名 | 说明 | 来源单元 |
|------|------|--------|
| `registry_config` | 注册中心连接配置（type / server_addr / auth / register_self），密码 AES 加密 | REG-1 |

同时 `sys_config` 追加 4 个 KV：`registry.self.service_name`（默认 POWERGATEWAY）、`registry.self.ip.override`、`registry.heartbeat.interval.seconds`（默认 5）、`registry.heartbeat.fail.threshold`（默认 3）。
