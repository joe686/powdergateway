## 两大核心模块

### 模块一：报文格式转换

面向接口报文的格式互转与字段加工，配置结果保存为可复用模板。

- **格式转换**：JSON ↔ XML ↔ CSV ↔ FormData（可扩展 SOAP、Protobuf）
- **字段映射**：拖拽方式配置源字段 → 目标字段的对应关系
- **字段处理**：截取、补位、类型转换、四则运算等
- **渠道模板**：识别渠道标识字段，按渠道应用不同转换模板
- **模板管理**：模板的保存、复用、编辑、删除

```
请求报文 → 格式解析 → 字段映射 → 字段处理 → 渠道模板匹配 → 输出报文
                                                      ↓
                                               模板持久化（配置库）
```

### 模块二：可视化接口开发（CRUD）

通过界面配置生成 SQL 并对接目标数据库，支持高并发访问。

- **数据库连接**：可视化添加 MySQL/Oracle/PostgreSQL 数据源
- **CRUD 配置**：
  - 查询：多表关联、字段筛选、条件构建、数据加工
  - 新增：请求字段校验、自定义字段、计算字段
  - 修改：唯一条件强制、修改前后数据对比
  - 删除：安全确认、防批量误删、唯一条件校验
- **分库分表**：独立菜单配置分片规则
- **缓存策略**：双层缓存（Caffeine 本地 + Redis 分布式），支持缓存预热和防击穿
- **字段公式**：可复用的复杂查询条件和计算逻辑
- **数据审计**：所有增删改操作异步写入独立审计库（含操作人、IP、时间、SQL、执行结果）

```
可视化配置 → SQL生成 → 分库分表解析 → 数据库执行 → 字段加工 → 缓存写入 → 响应
                                              ↓（异步）
                                        审计日志 → 审计库
```

## 前端菜单结构

```
首页
  └─ 系统概览 / 新手引导
接口转换配置（模块一）
  └─ 报文格式转换 / 字段转换配置 / 字段加工配置 / 渠道模板管理 / 转换模板管理
可视化接口开发（模块二）
  └─ 数据库连接管理 / 表结构管理 / 接口开发 / 分库分表配置
  └─ 常用字段公式管理 / 缓存查询管理 / 接口配置导航（9步向导）
系统管理
  └─ 日志管理 / SQL留存管理 / 性能统计 / 用户权限管理 / 系统配置
辅助工具
  └─ 接口文档（Swagger） / 报文调试 / 插件管理
```

菜单支持角色权限控制（admin/user/readonly）和开关控制（日志菜单等）。

## 配置库核心表（9张）

| 表名 | 说明 |
|------|------|
| `sys_user` | 用户表，BCrypt 密码 |
| `convert_template` | 转换模板，含 mapping_rule、process_rule（JSON） |
| `channel_config` | 渠道配置，渠道码 + 识别字段 + 关联模板 |
| `port_route` | 端口分发路由，`channel_code`+`port_address`+`request_template_id`（A→B）+`response_template_id`（B→A） |
| `db_connection` | 数据源，密码 AES 加密，支持 MySQL/Oracle/PG |
| `interface_config` | 接口配置，config_json 存全量配置，status: draft/published/disabled |
| `shard_config` | 分库分表规则 JSON |
| `field_formula` | 可复用字段公式 |
| `sys_config` | 全局 KV 配置（缓存TTL、日志留存天数等） |

审计库单独建：`sql_audit_log`（含 before_snapshot、op_ip、sql_text）。
