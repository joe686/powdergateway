# AUX-2 首页系统概览 设计文档

- 日期：2026-05-20
- 单元：AUX-2
- 状态：设计完成，待实施
- 前置依赖：SYS-2（性能统计）、M2-7（接口发布与统一执行入口）、M2-10（双层缓存）、SYS-3（菜单权限）

## 1. 背景

PowerGateway 当前首页 `views/DashboardView.vue` 是一个占位页（4 张零值卡片 + 欢迎区），登录后默认跳转到 `/dashboard`，三角色（admin/operator/viewer）均有访问权限。

AUX-2 要把它改造为真正的"系统概览"，把以下能力可视化呈现：

- 接口规模（总数、各状态分布）
- 今日/本周/本月调用量与成功率
- 调用趋势（折线图）
- 操作类型分布（柱状图）
- 慢接口排行
- 缓存命中率
- 未处理告警提示
- 快捷入口

## 2. 范围

### 2.1 包含

| 区块 | 内容 | 数据来源 |
|------|------|---------|
| 5 张数字卡片 | 总接口数、已发布数、今日调用次数、今日成功率、缓存命中率 | `interface_config` + `perf_stat` + `QueryCacheManager` |
| 调用趋势折线图 | today 维度按小时 / week/month 维度按天，含 successCounts/failCounts 双线 | `PerfStatService.summary(dimension)` |
| opType 柱状图 | 维度窗口内各 SELECT/INSERT/UPDATE/DELETE 调用量 | 新增 `PerfStatMapper.groupByOpType` |
| 接口状态饼图 | draft / published / disabled 数量占比 | `interface_config` GROUP BY status |
| TOP 5 慢接口表格 | 维度窗口内平均耗时倒序前 5，含接口名 / 平均耗时 / 调用次数 | 新增 `PerfStatMapper.topSlowInterfaces`（LEFT JOIN `interface_config`） |
| 未处理告警列表 | resolved=0 的最近 10 条告警，红色高亮 | 新增 `PerfAlertMapper.selectUnresolvedLatest` |
| 快捷入口 | 新建接口（→ /interface/wizard）、格式转换（→ /convert/format）、报文调试（→ /tools/debug） | 路由跳转 |
| 维度切换 | today / week / month 三档单选，切换后所有区块刷新 | URL 参数 `dimension` |
| 自动轮询 | 30s 拉取一次；标签页隐藏暂停，可见恢复时立即拉一次 | 前端 setInterval + Page Visibility API |

### 2.2 不含

- 自定义时间范围选择（YAGNI，当前需求只要三档维度）
- 告警处理交互（标记 resolved、批量处理）— SYS-2 范围
- 接口详情下钻（点 TOP5 跳详情页）— 暂不做，仅展示
- 数据导出（首页不需要，统计页 SYS-2 已有）
- 后端缓存：首页 API 不做服务端缓存，依赖 30s 轮询本身限频，逻辑简单

#### HomeOverviewService 关键逻辑

```java
public HomeOverviewDTO getOverview(String dimension) {
    // 1. 解析 dimension → [from, to) 窗口
    TimeWindow window = resolveWindow(dimension);

    // 2. 接口规模：BaseMapper.selectCount + GROUP BY status
    InterfaceStatsDTO interfaceStats = computeInterfaceStats();

    // 3. 调用统计：perfStatMapper.statBetween(from, to)
    CallStatsDTO callStats = computeCallStats(window);

    // 4. 调用趋势：复用 perfStatService.summary(dimension)
    CallTrendDTO callTrend = computeCallTrend(dimension);

    // 5. opType 柱图：perfStatMapper.groupByOpType(from, to)
    List<OpTypeCountDTO> opTypeDist = computeOpTypeDistribution(window);

    // 6. TOP5 慢接口：perfStatMapper.topSlowInterfaces(from, to, 5)
    List<SlowInterfaceDTO> topSlow = computeTopSlow(window, 5);

    // 7. 未处理告警：perfAlertMapper.selectUnresolvedLatest(10)
    List<PerfAlert> alerts = perfAlertMapper.selectUnresolvedLatest(10);

    return new HomeOverviewDTO(interfaceStats, callStats, callTrend,
                               opTypeDist, topSlow, alerts);
}

private CallStatsDTO computeCallStats(TimeWindow w) {
    Map<String, Object> stat = perfStatMapper.statBetween(w.from, w.to);
    long total = getLong(stat, "total");
    long failCount = getLong(stat, "failCount");
    BigDecimal successRate = total == 0
        ? BigDecimal.ZERO
        : BigDecimal.valueOf((total - failCount) * 100.0 / total).setScale(1, RoundingMode.HALF_UP);
    long avgMs = getLong(stat, "avgMs");
    BigDecimal cacheHitRate = computeCacheHitRate();   // 见下
    return new CallStatsDTO(total, successRate, avgMs, cacheHitRate);
}

private BigDecimal computeCacheHitRate() {
    // 遍历 cacheEnabled=1 的接口，对每个调用 queryCacheManager.getStats() 求和
    List<InterfaceConfig> enabled = interfaceConfigMapper.selectList(
        new LambdaQueryWrapper<InterfaceConfig>().eq(InterfaceConfig::getCacheEnabled, 1));
    long totalHit = 0, totalMiss = 0;
    for (InterfaceConfig cfg : enabled) {
        CacheStatDTO s = queryCacheManager.getStats(cfg.getId());
        totalHit  += s.getHitCount();
        totalMiss += s.getMissCount();
    }
    long sum = totalHit + totalMiss;
    if (sum == 0) return null;        // 前端显示 "—"
    return BigDecimal.valueOf(totalHit * 100.0 / sum).setScale(1, RoundingMode.HALF_UP);
}
```

#### 时间窗口解析

```java
private TimeWindow resolveWindow(String dimension) {
    LocalDateTime to = LocalDate.now().plusDays(1).atStartOfDay();
    LocalDateTime from;
    switch (dimension != null ? dimension : "today") {
        case "week":  from = LocalDate.now().minusDays(6).atStartOfDay(); break;
        case "month": from = LocalDate.now().minusDays(29).atStartOfDay(); break;
        default:      from = LocalDate.now().atStartOfDay();   // today 或非法值
    }
    return new TimeWindow(from, to);
}
```

#### 新增 Mapper SQL

```java
// PerfStatMapper
@Select("SELECT op_type AS opType, COUNT(*) AS count " +
        "FROM perf_stat WHERE stat_time >= #{from} AND stat_time < #{to} " +
        "GROUP BY op_type ORDER BY count DESC")
List<Map<String, Object>> groupByOpType(@Param("from") LocalDateTime from,
                                        @Param("to")   LocalDateTime to);

@Select("SELECT p.interface_id AS interfaceId, " +
        "       i.name          AS interfaceName, " +
        "       ROUND(AVG(p.cost_ms), 0) AS avgCostMs, " +
        "       COUNT(*)        AS callCount " +
        "FROM perf_stat p LEFT JOIN interface_config i ON p.interface_id = i.id " +
        "WHERE p.stat_time >= #{from} AND p.stat_time < #{to} " +
        "GROUP BY p.interface_id, i.name " +
        "ORDER BY avgCostMs DESC " +
        "LIMIT #{limit}")
List<Map<String, Object>> topSlowInterfaces(@Param("from") LocalDateTime from,
                                            @Param("to")   LocalDateTime to,
                                            @Param("limit") int limit);

// PerfAlertMapper
@Select("SELECT * FROM perf_alert WHERE resolved = 0 " +
        "ORDER BY check_time DESC LIMIT #{limit}")
List<PerfAlert> selectUnresolvedLatest(@Param("limit") int limit);
```

> **大小写兼容**：H2 返回大写别名，MySQL 返回小写，沿用 `PerfStatService.getStr / getLong` 的不区分大小写读法。

#### 轮询机制

```js
let timer = null
function startPolling() {
  if (timer) return
  timer = setInterval(reload, 30_000)
}
function stopPolling() {
  if (timer) { clearInterval(timer); timer = null }
}
function onVisibilityChange() {
  if (document.visibilityState === 'visible') {
    reload()                  // 恢复时立即拉一次
    startPolling()
  } else {
    stopPolling()
  }
}
onMounted(() => {
  reload()
  startPolling()
  document.addEventListener('visibilitychange', onVisibilityChange)
})
onUnmounted(() => {
  stopPolling()
  document.removeEventListener('visibilitychange', onVisibilityChange)
})
```

#### 防御性 Null 处理

按"前端已知 Bug 模式"备忘：
- `cacheHitRate === null` → 卡片显示 `—`
- `interfaceName` 缺失 → 显示 `接口#${id}` 或 `—`
- `activeAlerts.length === 0` → 列表区显示 `el-empty 暂无未处理告警`，不显示红色徽标

## 7. 错误处理

| 场景 | 处理 |
|------|------|
| dimension 非法 | 静默回退 today，不报错 |
| Redis 未启动（测试环境） | `QueryCacheManager.getStats` 返回 0/0；`computeCacheHitRate` 返回 null |
| 个别 mapper 查询返回 null | 用 `Optional.ofNullable().orElse(0L/Collections.emptyList())` 兜底 |
| 前端请求失败 | `ElMessage.error` 提示，保留上一次成功的数据；30s 后自动重试 |
| 标签页隐藏 | 暂停轮询；可见时立即补一次再启动轮询 |

1. `/api/home/overview` 接口在 4 个维度下（today/week/month/invalid）均返回 200 + 完整字段

6. 全量回归测试通过（`mvn test`），新增至少 14 条 AUX-2 测试，全部绿
