## 四、维护约定

- **常态巡检**：每次发新版本前扫一遍本文件，勾掉已落地条目、把新识别的缺陷/待办追加进对应池
- **不重复登记**：若一条待办已同步到 `03-开发/任务计划/` 下的详细任务分解，正文里加 `→ 见任务计划/YYYY-MM-DD-XXX.md`，避免两处内容漂移
- **及时删陈旧**：明确决定不做的条目，从"待评审"直接进"已拒绝"并写清原因，不长期挂着占位

## 五、快速命令速查

```bash
# 打 patch tag
git tag -a v0.1.1 -m "PowerGateway v0.1.1 · <一句话总结>"
git push origin master v0.1.1

# 本地打便携版（Win）
bash scripts/build/build-portable.sh --platform win-x64 --version 0.1.1

# 本地打标准包
bash scripts/build/build-standard.sh --version 0.1.1

# 本地冒烟
bash scripts/ci/verify-artifacts.sh dist/powergateway-standard-0.1.1.zip

# 后端全量测试
cd backend && mvn test

# 前端 build + 单测
cd frontend && npm run build && npx vitest run
```
