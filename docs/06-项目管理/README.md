# 项目管理

本章节用于**版本基线声明、路线图规划、待办与缺陷池管理**，是单人开发模式下项目管理的中枢。

## 文档索引

| 文档 | 用途 | 变更频率 |
|------|------|---------|
| **[反馈簿.md](./反馈簿.md)** | **用户唯一反馈入口 · FB-XXX 全量流转 · Claude 走 `pg-feedback-flow` skill 处理** | **高（持续追加与回写）** |
| [v0.1.0-基线.md](./v0.1.0-基线.md) | v0.1.0 首个正式版本基线声明：构成清单、测试规模、制品 tag | 只增不改（发新版时新建 `vX.Y.Z-基线.md`） |
| [路线图.md](./路线图.md) | 分版本演进计划：v0.1.x 补丁 / v0.2.0 / v0.3.0 / v1.0 / v2+ | 每次版本发布或规划调整时更新 |
| [待办与缺陷池.md](./待办与缺陷池.md) | Backlog（未做功能）+ 缺陷池 + 变更申请模板 | 持续维护 |

## 与其他章节的关系

```
docs/
├── 01-需求/                产品需求 · 稳定文档，规格描述
├── 02-设计/                架构与详细设计 · 落地方案
├── 03-开发/
│   ├── 变更记录.md         CHG-XXX 归档 · 已交付后的范围变更
│   ├── 开发计划.md         阶段与单元的总览表
│   └── 任务计划/           单元级 TDD 任务分解
├── 04-测试/                TDD 规范
├── 05-制品发布/            SKU 产品说明书
└── 06-项目管理/            ← 本章：基线 / 路线图 / 待办
```

**分工原则**：

- `01-需求` / `02-设计` 回答"要做什么、怎么做"
- `03-开发` 回答"什么时候做、变更了什么"
- `04-测试` 回答"怎么验收"
- `05-制品发布` 回答"给谁用、怎么部署"
- **`06-项目管理` 回答"现在是什么版本、下一步做什么、还欠什么"**

## 工作节律（单人开发模式）

1. **常态迭代**：Bug 修复 / 小功能 → 直接 commit + `git tag -a vX.Y.Z` → 更新[路线图](./路线图.md)"已发布"表
2. **范围变更**：按根 `CLAUDE.md` 的变更规约三步走（`变更记录.md` + 需求 + 开发计划），并把"待办"从[待办池](./待办与缺陷池.md)勾掉
3. **版本切换**：发新版时新建 `vX.Y.Z-基线.md`，把[路线图](./路线图.md)对应版本从"规划中"移入"已发布"
