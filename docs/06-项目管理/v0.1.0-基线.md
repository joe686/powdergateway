# v0.1.0 基线

> **首个正式版本 · 定基时间 2026-07-22**
> Git tag：[`v0.1.0`](../../) → commit `2dbfc26`
> 状态：**已发布**（本地打包 + git tag 标记版本，不走 GitHub Actions）

---

## 一、基线定义

v0.1.0 = **28 个交付单元全绿 + CHG-016 ~ CHG-027 全部归档 + 555 后端测试用例全绿 + 前端 build 通过 + pg-testkit compile 通过**。

一旦本文档定稿，v0.1.0 基线内容**只读**。后续任何修改：

- Bug 修复 → 递进到 v0.1.x
- 小功能新增 → 递进到 v0.2.0
- 参见 [路线图](./路线图.md)

## 二、基线构成清单

### 阶段一 · 基础设施（P0，4 单元）

| 单元 | 交付内容 |
|------|---------|
| **P0-1** | 全局异常处理（`BusinessException` + `MethodArgumentNotValidException` + 兜底）、统一响应体 `Result<T>`、健康检查、H2 测试基础设施 |
| **P0-3** | 配置库 8 张表 DDL + MyBatis-Plus 实体类 + Mapper |
| **P0-4** | Sa-Token 认证、登录接口 `POST /api/auth/login`、`SaTokenConfig` |

> P0-2 由 P0-3 / P0-4 合并覆盖，未单列。

### 阶段二 · 接口转换（M1，7 单元）

| 单元 | 交付内容 |
|------|---------|
| **M1-1** | `FormatConverter`：JSON ↔ XML ↔ CSV 等 12 种互转 |
| **M1-2** | 字段映射配置：`mapping_rule` JSON 存 `convert_template`，`ConvertController` 保存/预览映射 |
| **M1-3** | `FieldProcessor`：策略模式字段加工引擎（Trim/Pad/Substring/Case/TypeCast） |
| **M1-4** | 渠道配置：`ChannelConfigController/Service`，运行时按渠道字段自动选模板 |
| **M1-5** | 转换模板 CRUD：`TemplateController/Service`，含分页搜索、版本留存（`is_latest`） |
| **M1-6** | 报文转换串联接口 `POST /api/convert`：格式转换 → 字段映射 → 字段加工全链路，Redis 缓存模板 |
| **M1-7** | 端口分发路由：`port_route` 表 + `POST /api/dispatch`，双向转换 + `HeaderConfigMerger` + `CharsetConverter` |

### 阶段三 · 可视化接口（M2 前段，8 单元）

| 单元 | 交付内容 |
|------|---------|
| **M2-1** | 数据库连接管理：MySQL/Oracle/PG，密码 AES-128，测试连通 |
| **M2-2** | 表结构查询：`DatabaseMetaData` + Redis 缓存（TTL 24h）+ 手动刷新 + Excel 导出 |
| **M2-3** | 查询接口配置：`QueryBuilder`、4 步向导 |
| **M2-4** | 插入接口配置：`InsertBuilder`、`DataSourceResolver`、`ColumnValidator`、多表 JDBC 手动事务 |
| **M2-5** | 修改接口配置：`UpdateBuilder`，强制唯一条件校验，修改前快照 |
| **M2-6** | 删除接口配置：`DeleteBuilder`，待删数据预览，批量删除保护开关 |
| **M2-7** | 接口发布：状态流转 `draft → published → disabled`，统一执行入口 `/api/exec/{id}`，`OpenApiDynamicCustomizer` Swagger 动态注册 |
| **M2-9** | SQL 审计日志：`@AuditLog` AOP + LinkedBlockingQueue 异步 + 每天清理 |

### 阶段四 · 高级功能（M2 后段 + SYS，6 单元）

| 单元 | 交付内容 |
|------|---------|
| **M2-10** | 双层缓存 Caffeine → Redis → DB + 分布式锁防击穿 + 命中统计 |
| **M2-8** | 分库分表：`ShardRouter` MODULO/RANGE 路由，`resolveSharding()` 在 4 种 exec 方法中替换主表 + dbConnectionId |
| **SYS-1** | 日志管理：`@SysLogRecord` AOP 写 `sys_log`，双 Tab（操作日志 + SQL 审计），Excel 导出，`SysLogArchiveJob` 归档到 history |
| **SYS-2** | 性能统计：`@PerfStat` AOP，折线图 + 柱状图，`PerfAlertJob` 定时告警 |
| **SYS-3** | 用户权限管理：三角色（admin / user / readonly）+ `MenuPermission` 白名单 + 动态菜单 + Sa-Token `@SaCheckRole` |
| **SYS-4** | 系统配置：`SysConfigService` `@PostConstruct` 预热 + `ApplicationEvent` 热更新广播 |

### 阶段五 · 体验优化（SYS-5 + AUX，3 单元）

| 单元 | 交付内容 |

| **SYS-5** | 接口 9 步向导：纯前端 `InterfaceWizard.vue`（867 行），Pinia + localStorage 保存中间状态 |
| **AUX-1** | 报文调试工具：`MessageDebug.vue`，格式转换 + 接口调用双模式 |
| **AUX-2** | 首页系统概览：5 卡片 + 3 图表 + TOP5 表 + 告警列表 + 维度切换 + 30s 轮询 |

## 六、v0.1.0 已知不含（明确延后）

以下能力**不在**基线内，是路线图未来版本的范围（详见 [路线图.md](./路线图.md)）：

- SQLite 分支的 Maven profile 与完整回归（v0.2.0）
- pg-testkit Faker 10 万条数据 + 完整 10 表 DDL + Mock 规则持久化（v0.2.0）
- Oracle / OceanBase 驱动预置 + SQL 兼容性回归（v0.3.0 / REL-1B 子任务）
- Docker 部署形态（v0.3.0 或 v1.0）
- Windows exe 安装包（用户明确不做）
- ARM 架构（v2+）
- Windows Authenticode 数字签名（需商业证书，暂不做）
