## 规划中

### v0.1.x · 补丁通道（持续）

**目标**：修复 v0.1.0 交付后暴露的 bug、小体验问题，不新增功能。

**触发条件**：出现 Bug 或视觉/交互瑕疵。

**发布节律**：一 bug 一 commit → 累计 3~5 个后打 patch tag（`v0.1.1` / `v0.1.2` ...）；或紧急问题独立打 tag。

**已识别修复候选**：见 [待办与缺陷池 · 缺陷池](./待办与缺陷池.md#二缺陷池)。

**新增功能一律不进 patch 通道**，走 v0.2.0。

## 版本命名约定

采用 [语义化版本](https://semver.org/lang/zh-CN/)：

- `vMAJOR.MINOR.PATCH`
- **MAJOR**：不兼容的 API/配置变更（v1.0 之前允许 v0.x 破坏性变更）
- **MINOR**：新功能 / 新单元交付
- **PATCH**：Bug 修复 / 视觉修正 / 文档修正

**打 tag 命令**：

```bash
git tag -a v0.1.1 -m "PowerGateway v0.1.1 · <一句话总结>"
git push origin v0.1.1
```

**打 tag 前 checklist**：

- [ ] 涉及的 CHG-XXX 都归档到 [变更记录.md](../03-开发/变更记录.md)
- [ ] [待办与缺陷池.md](./待办与缺陷池.md) 对应条目已勾掉
- [ ] 后端 `mvn test` 全绿
- [ ] 前端 `npm run build` 通过
- [ ] 更新本文件"已发布"表
